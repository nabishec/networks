# Count-to-infinity problem

## TL;DR
Болезнь distance-vector алгоритмов: при разрыве канала маршрутизаторы могут «обманываться» через циклические обходы и **постепенно** увеличивать оценку расстояния шаг за шагом до **бесконечности** (или верхнего предела вроде 16 в RIP). Конвергенция занимает много раундов. Решения: split horizon, poison reverse, hold-down timers — палиативные.

## Какую проблему решает
Объясняет, **почему DV-протоколы плохо справляются с обрывами линков** и **почему интернет в 1990-х перешёл с RIP на OSPF (link-state)**. Понимание count-to-infinity — обязательно для проектирования IGP.

## Как работает

**Сценарий (RIP, 4 узла A-B-C-D в цепочке):**

Шаг 0 (норма):
- A→D = 3 (через B)
- B→D = 2 (через C)
- C→D = 1 (через D напрямую)
- D себя знает = 0.

Канал C↔D отказал. C должен «понять», что D недостижим.

Шаг 1: C видит, что D больше не сосед. Но **B говорит C: «у меня D = 2»**. C поверил, обновил: `D через B = 3`.

Шаг 2: B слышит обновление от C: `D = 3 через ?`. B думает: я раньше шёл через C, теперь C говорит D = 3. Значит, мой путь через C = 4.

Шаг 3: C слышит от B: `D = 4`. Через B путь = 5. Обновил.

И так далее: **C↔B бесконечно увеличивают друг другу метрику**, веря в обходной путь.

**Если предела нет — count-to-infinity** буквально.

**В RIP:** установлен max=15; при достижении 16 объявляется недостижимость. Конвергенция за несколько update-циклов (~минуты).

## Пример
В реальной сети с 50 RIP-маршрутизаторами обрыв одного канала может приводить к 5–15 минутам нестабильности маршрутов, петлям (пакеты ходят по кругу, дохнут от TTL), потерям.

OSPF (link-state) сходится за **секунды** в той же ситуации — каждый маршрутизатор пересчитывает Дейкстру на актуальной топологии.

## Связи
- **Базируется на:** [[Distance Vector Routing]] (это её болезнь).
- **Используется в:** объяснение мотивации [[Link State Routing]] и OSPF.
- **Соседи по уровню:** **split horizon, poison reverse** — палиативные техники в DV.
- **Противопоставляется:** [[Link State Routing]] — там этой проблемы нет (полная топология, Дейкстра сходится за один проход).

## Подводные камни
- **Split horizon** (не объявлять соседу путь, который через него) и **poison reverse** (объявлять с ∞) уменьшают, но **не устраняют** проблему — есть сценарии с тремя+ узлами в петле.
- В современных сетях DV-протоколы (RIP) **почти не используются** именно из-за count-to-infinity. EIGRP с DUAL — исключение.
