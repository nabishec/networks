# Datagram subnet vs virtual-circuit subnet

## TL;DR
Две парадигмы реализации сетевого уровня. **Datagram**: каждый пакет независим, в заголовке полный destination, маршрутизатор решает куда отправить per-packet (IP). **Virtual circuit**: при установлении соединения резервируется путь, пакеты несут короткий VC-ID; маршрутизатор по таблице VC переключает (ATM, MPLS-LSP, X.25).

## Какую проблему решает
Та же дилемма, что и [[Соединение vs без соединения]] на L4, — но **на сетевом уровне**: гибкость отдельных пакетов vs предсказуемость пути. Влияет на отказоустойчивость, QoS, размер state в маршрутизаторах.

## Как работает

| | Datagram (IP) | Virtual circuit (ATM/MPLS) |
|---|---|---|
| Setup | нет | есть фаза установления |
| Адрес в пакете | полный DST | короткий VC-ID |
| State в маршрутизаторе | таблица prefix→next-hop, **не per-flow** | per-VC state |
| Отказ маршрутизатора | пакеты переадресуются | VC рвётся, нужен реустановка |
| QoS | best-effort или DSCP | заложено в setup |
| Примеры | IP интернета | ATM, Frame Relay, MPLS, SDN flows |

## Пример
- **IP:** скачиваешь файл — каждый пакет может пойти разным маршрутом, переупорядочение и потери возможны (TCP исправит).
- **MPLS-LSP:** провайдер заводит «трубу» от Москвы до Питера; все пакеты, маркированные label X, идут по ней. Стабильно, предсказуемо, но при отказе линка — переустановка.

## Подводные камни
- IP — datagram, но **внутри** оператора часто MPLS — гибрид.
- TCP/QUIC поверх IP создают «virtual circuit» на L4 — это не противоречит datagram'у на L3.
