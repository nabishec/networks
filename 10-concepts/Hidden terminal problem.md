# Hidden terminal problem (проблема скрытой станции)

## TL;DR
Два узла **не слышат друг друга** (далеко по радио или есть препятствие), но **оба слышат точку доступа (AP)**. Каждый делает carrier sensing, видит свободный эфир и передаёт → у AP **коллизия**. CSMA/CA не помогает — она основана на способности узлов слышать друг друга. Главная фундаментальная слабость беспроводного MAC; смягчается через [[RTS/CTS]].

## Какую проблему решает
Описывает реальный сценарий, где даже идеальный CSMA/CA даёт сбои. Понимание hidden terminal — обязательно для проектирования Wi-Fi-сетей: расположение AP, использование RTS/CTS, выбор каналов.

## Как работает

**Сценарий:**
- Узел A слышит AP, но не слышит C.
- Узел C слышит AP, но не слышит A.
- A передаёт → C ничего не слышит, начинает свою передачу → у AP коллизия.

```mermaid
flowchart LR
  A((Узел A)) -.слышит.- AP((AP))
  C((Узел C)) -.слышит.- AP
  A -.НЕ слышит C.- C
  A --передача-->|✗| AP
  C --передача-->|✗| AP
  Note[у AP — коллизия]
```

**Почему так бывает:**
- Расстояние: A и C на разных краях Wi-Fi-зоны, AP в центре. Радиус слышимости узлов меньше радиуса AP (более мощного передатчика).
- Препятствия: стена между A и C прозрачна для AP (он на потолке), но непроницаема для горизонтального сигнала.
- Геометрия в больших офисах с высокими потолками — типична.

## Пример
**Большая комната с офисом open-space:**
- AP на потолке в центре.
- Сотрудник A в одном углу, сотрудник C в противоположном.
- Прямой видимости нет (мебель), но AP видят оба.
- Когда A качает большой файл и C открывает страницу одновременно → у AP «фрейм A искажён, фрейм C искажён», оба ретрансмитят.

**Решение:** [[RTS/CTS]]. A перед большим transfer'ом шлёт RTS («Request to Send») на AP. AP отвечает CTS («Clear to Send»). C **слышит CTS от AP** (хотя не слышит RTS от A) → знает «канал занят» → молчит.

## Связи
- **Базируется на:** [[CSMA/CA]] — описывает сценарий, где она не справляется; [[802.11 — Wi-Fi архитектура]].
- **Используется в:** [[RTS/CTS]] — частичное решение; [[NAV — Network Allocation Vector]] — механизм виртуального резервирования.
- **Соседи по уровню:** [[Exposed terminal problem]] — обратная по характеру проблема.
- **Противопоставляется:** проводной CSMA/CD — там все слышат друг друга через общую среду, hidden terminal не существует.

## Подводные камни
- RTS/CTS **уменьшает** проблему, но **не устраняет**: сама фаза RTS уязвима к коллизиям. Кроме того, RTS/CTS — overhead, имеет смысл только для длинных фреймов. По умолчанию Wi-Fi включает его выше определённого порога (часто 2347 байт, т.е. практически отключён).
- Hidden terminal часто **диагностируется** как «странные тормоза Wi-Fi» — клиенты видят полный сигнал к AP, но реальная скорость низкая из-за коллизий, которых они не наблюдают.
- В современном Wi-Fi (OFDMA, beamforming) AP может физически разносить пользователей по поднесущим/направлениям — частично смягчает hidden terminal.
