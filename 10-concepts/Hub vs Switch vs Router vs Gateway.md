# Hub vs Switch vs Router vs Gateway

## TL;DR
Четыре типа устройств, **каждое на своём уровне**: **hub** = L1 (повторитель битов), **switch** = L2 (по MAC), **router** = L3 (по IP), **gateway** = L7 (преобразование между протоколами). Чем выше уровень — тем больше «соображает» устройство, но и медленнее, и дороже. В обиходе термины путаются — особенно «домашний роутер» обычно совмещает switch+AP+router+NAT+DHCP.

## Какую проблему решает
Понимание, **на каком уровне** работает устройство, объясняет:
- Что оно может «видеть» (биты, фреймы, пакеты, payload).
- Что меняет в проходящих пакетах.
- Какие проблемы решает (коллизии, маршрутизация, межпротокольный gateway).
- Где его место в архитектуре сети.

Без этой иерархии разговор о сетях разъедется в шум маркетинговых терминов.

## Как работает

| Устройство | OSI Уровень | На что смотрит | Что делает | Пример |
|---|---|---|---|---|
| **Hub** | L1 | биты | повторяет вход во все остальные порты | устаревший, заменён switch'ем |
| **Switch (bridge)** | L2 | MAC-адрес | пересылает фрейм только в нужный порт по таблице MAC→порт | офисный коммутатор |
| **Router** | L3 | IP-адрес | пересылает пакет между сетями, обновляет L2-заголовок | домашний роутер, BGP-маршрутизатор |
| **Gateway** | L4–L7 | payload, протокол | преобразует между протоколами/представлениями | API gateway, mail gateway, NAT (border) |

```mermaid
flowchart LR
  Hub[Hub L1<br/>повторяет биты] --> Switch[Switch L2<br/>читает MAC]
  Switch --> Router[Router L3<br/>читает IP]
  Router --> Gateway[Gateway L7<br/>понимает приложение]
```

**Что меняется в проходящем пакете:**

| Уровень | Меняется на устройстве? |
|---|---|
| L1 биты | hub — копия; switch и выше — пересоздание сигнала |
| L2 MAC | switch — нет (switch не меняет L2-адреса в одной сети); router — **да** (новый MAC src/dst) |
| L3 IP | router — TTL уменьшается, checksum пересчитывается; адрес — **не меняется** (кроме NAT) |
| L4 порты | router — **не меняет** (кроме NAT); gateway может |
| L7 payload | gateway — **может** (например, преобразование IMAP↔POP3) |

## Пример
**Домашний «роутер» от провайдера** — это в одном корпусе:
- **Switch** на 4 LAN-порта (L2).
- **Wi-Fi AP** (L1+L2 для радио).
- **Router** между LAN и WAN (L3).
- **NAT** (L3+L4, преобразует адреса/порты).
- **DHCP-сервер** (L7).
- **DNS-relay** (L7).
- Часто **firewall** (L3+L4).

Поэтому строго отвечать «это что» — некорректно; правильный ответ зависит от того, какую функцию рассматриваем.

**Хаб против switch'а:**
- 5 устройств на 10BASE-T хабе: ~2 Мбит/с на каждое (10 / 5 при коллизиях).
- 5 устройств на 1000BASE-T switch'е: 1 Гбит/с на каждое до bottleneck.

## Подводные камни
- **Хабы** в новых установках не встречаются — даже cheap-уровневые устройства сегодня switch'и. Но термин «hub» жив в маркетинге («USB hub»).
- **Layer 3 switch** — это switch с маршрутизирующим ASIC: умеет L3 со скоростью L2. В дата-центрах это **доминирующий** тип устройства.
- **Gateway** — самый размытый термин. В L3 гoворят «default gateway» = адрес роутера для исходящих. В L7 — application gateway / API gateway. Контекст важен.
- **NAT-роутер** строго не L3-only: он смотрит на TCP/UDP-порты (L4). Поэтому в академии NAT иногда называют «middlebox» — нарушение чистого деления уровней.
