# IXP и пиринг

## TL;DR
**IXP** (Internet Exchange Point) — физическая точка (datacenter), где много операторов подключаются к одному L2-fabric (большому switch'у) и обмениваются трафиком напрямую. **Пиринг** — экономическая модель: «давай прокачивать друг другу трафик без оплаты» (settlement-free) или с оплатой (paid peering). Альтернатива — [[Транзитная сеть|транзит]] (платишь upstream'у). IXP радикально снижают стоимость и задержку трафика.

## Какую проблему решает
Если нет IXP, операторы должны идти через transit-провайдеров → дополнительные хопы и плата. С IXP — прямой L2-обмен, локальный трафик не покидает страну/город.

## Как работает

**Архитектура IXP:**
- Большой Ethernet/MPLS-fabric в нейтральном datacenter.
- Каждый оператор подключает свой router в этот fabric (10/100/400 Гбит/с порт).
- BGP-сессии между операторами — частные (bilateral) или через **route server** (multilateral).
- IXP сам **не маршрутизирует** — только L2-меняет фреймы. Маршрутизация остаётся у участников.

**Известные IXP:**
- **DE-CIX** (Frankfurt) — крупнейший в Европе, ~10 Тбит/с пик.
- **AMS-IX** (Amsterdam).
- **LINX** (London).
- **MSK-IX** (Москва) — крупнейший в России.
- **Equinix IX** — глобальная сеть.

**Виды peering'а:**
- **Settlement-free peering (SFP):** бесплатно. Обычно для «равных» операторов (eyeball + content network).
- **Paid peering:** один платит другому. Чаще когда трафик асимметричен (Netflix → Comcast).
- **Bilateral / multilateral:** prearranged между парами или через route server.

**Пиринговая политика операторов** — публикуется в **PeeringDB**:
- «Open peering» — пиримся со всеми.
- «Selective» — по критериям.
- «Restrictive» — только с large carriers.

## Пример
**Российский провайдер связан с IXP MSK-IX:**
- Один порт 100 Гбит/с во Frankfurt'е.
- Пиринг с Cloudflare, Google, Akamai → их CDN cache локально.
- Трафик **не покидает Россию** (если не нужно) → ниже latency, не платит за transit.
- Bandwidth-savings миллионы $/год.

**Без IXP (или с малым числом):**
- Трафик из MSK к google.com мог бы идти через Tier-1 в Стокгольме → +30 мс задержки, +платный transit.

## Связи
- **Базируется на:** [[Транзитная сеть]] (экономика), [[BGP]] (механика обмена маршрутами).
- **Используется в:** [[Интернет — архитектура]] (топология современного интернета), CDN-deployment.
- **Соседи по уровню:** [[Пиринговый спор]] — конфликты в peering'е.
- **Противопоставляется:** transit-only — дороже, медленнее.

## Подводные камни
- **«Открытое пиринг»** — не значит «принять любого». Минимальные требования по trafic (например, > 1 Гбит) и архитектуре.
- **PeeringDB** — это самообъявленные политики; актуальность зависит от того, обновляет ли оператор.
- **De-peering** возможно (Cogent vs Hurricane Electric, периодические инциденты) — две сети взаимно отказываются → их клиенты не видят друг друга. Кратковременный «split internet».
- В России после 2022 г. многие международные операторы свернули peering → трафик идёт через transit с большей задержкой.
