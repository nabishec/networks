# MAC-адрес (MAC address, Hardware address)

## TL;DR
Уникальный 48-битный идентификатор сетевой карты, присваиваемый производителем. Записывается в шестнадцатеричном виде через `:` или `-`: `00:1B:44:11:3A:B7`. Состоит из двух частей: **OUI** (3 байта, идентификатор производителя) и **NIC** (3 байта, конкретное устройство). Используется в Ethernet, Wi-Fi, Bluetooth и любой L2-адресации.

## Какую проблему решает
На разделяемой среде (Ethernet, Wi-Fi) нужен **способ адресовать конкретное устройство** среди всех слышащих. IP-адрес для этого не подходит: он географический и может меняться (DHCP). MAC-адрес — **прибит к железу**, уникален на устройстве, и не меняется при переезде сети.

## Как работает

**Формат:**
```
00 : 1B : 44 : 11 : 3A : B7
└──── OUI ────┘ └─── NIC ───┘
   3 байта        3 байта
```

**OUI** (Organizationally Unique Identifier) выдаётся IEEE производителю: Cisco — `00:1B:54:...`, Apple — `00:1A:11:...`, Intel — `00:1B:21:...`. По OUI можно определить производителя устройства.

**Биты в первом байте:**
- **Бит 0 (LSB)**: 0 = unicast, 1 = multicast/broadcast. Например, `01:00:5E:...` — IP-multicast.
- **Бит 1**: 0 = universal (заводской), 1 = locally administered (вручную назначенный).

**Специальные:**
- **`FF:FF:FF:FF:FF:FF`** — broadcast: «всем в этой L2-сети».
- **`01:80:C2:00:00:0E`** — LLDP multicast.
- **`33:33:xx:xx:xx:xx`** — IPv6 multicast.

## Пример
- На ноутбуке `ifconfig` / `ip link` показывает MAC: `8c:85:90:0a:b2:e3`.
  - OUI `8c:85:90` → Apple. Это MacBook.
- `wireshark` фильтр `eth.src == 00:1b:44:11:3a:b7` — пакеты только от этого источника.
- **MAC randomization** в современных смартфонах: при подключении к Wi-Fi-сети генерируется случайный MAC (бит «locally administered») — приватность от трекинга по MAC. iOS и Android делают это по умолчанию с 2020 г.

## Связи
- **Базируется на:** [[Ethernet — IEEE 802.3]] (источник 48-битного формата), стандарт IEEE.
- **Используется в:** [[ARP]] (IP→MAC мэппинг), [[VLAN — IEEE 802.1Q]] (тегирование), [[802.11 — Wi-Fi архитектура]] (4-адресный фрейм), [[Bluetooth]] (тот же формат).
- **Соседи по уровню:** [[IP-адресация и CIDR]] — L3-аналог; разные роли.
- **Противопоставляется:** [[IP-адресация и CIDR|IP-адрес]] — географический, меняется в роуминге; MAC — стационарный (per-device), уникален.

## Подводные камни
- MAC можно **переопределить** программно (`ip link set dev eth0 address ...`) — он не «навечно сжжен» в железе. Используется и для приватности, и злоумышленниками для подмены.
- **MAC randomization** ломает корпоративный NAC (network access control), который раньше доверял MAC. Современные системы переходят на 802.1X / EAP.
- В L2 за пределами одной сети MAC **не виден** — на каждом маршрутизаторе он перезаписывается. По MAC из интернета вашего роутера определить нельзя.
- 48 бит = 281 трлн адресов. Глобально мало пересечений, но **в пределах одной сети** дубли катастрофичны. Локально-administrated random MAC ↔ universal MAC: возможны коллизии.
