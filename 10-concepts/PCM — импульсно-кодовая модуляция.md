# PCM — импульсно-кодовая модуляция (Pulse Code Modulation)

## TL;DR
Способ оцифровать аналоговый сигнал: в фиксированные моменты (8000 раз в секунду для голоса) **измеряем уровень** сигнала и записываем результат **числом фиксированной длины** (8 бит). Получаем 8000 × 8 = **64 кбит/с** на голосовой канал — фундамент всей цифровой телефонии (T1/E1, SONET, B-канал ISDN).

## Какую проблему решает
Аналоговый сигнал в магистрали — катастрофа: накапливается шум на каждом усилителе, помехи трудно отфильтровать, мультиплексирование сложное. Если вместо аналоговой волны передавать **числа**, шум перестаёт быть проблемой (числа точны), мультиплексирование становится тривиальным, появляется возможность сжатия и шифрования.

## Как работает

**Три шага:**
1. **Дискретизация (sampling):** по [[Теорема Найквиста|теореме Найквиста]] для восстановления сигнала с полосой B нужно ≥ 2B измерений в секунду. Голос — 4 кГц → 8000 выборок/с.
2. **Квантование:** каждое измерение округляется до ближайшего из **256 уровней** → 8 бит.
3. **Кодирование:** биты потоком идут в канал.

**8000 × 8 = 64 000 бит/с** — каноническая «голосовая ёмкость» в цифровой телефонии.

**Неравномерное квантование (компандирование):** ухо различает тихие звуки точнее, чем громкие. Поэтому квантование делают **логарифмическим** — мелкие шаги для тихих сигналов, крупные для громких. Два стандарта:
- **µ-law** (мю-закон) — Северная Америка, Япония.
- **A-law** — Европа, остальной мир.

Стандарт ITU-T **G.711** описывает оба варианта.

```
Аналог ─[Sample 8 kHz]─→ 8000 значений/с
       ─[Quantize 256 уровней]─→ 8 бит каждое
       ─[Code]─→ 64 кбит/с цифровой поток
```

## Пример
**Звонок Москва → Питер в PSTN:**
1. Твой голос (аналог 0.3–3.4 кГц) попадает в местную АТС.
2. Кодек на end office оцифровывает: 8000 выборок/с × 8 бит = 64 кбит/с.
3. 64-кбит-поток вкладывается в [[T1-E1 — иерархия PDH|T1/E1]] вместе с десятками других голосов (TDM).
4. Магистраль несёт цифровой поток до Питера.
5. На end office Питера обратное преобразование (декодек) → аналог в местной петле.

Каждое преобразование A→D→A добавляет лёгкое искажение, но шума не накапливается.

## Связи
- **Базируется на:** [[Теорема Найквиста]] — выбор частоты дискретизации.
- **Используется в:** [[PSTN — телефонная сеть]] (фундамент магистрали), [[T1-E1 — иерархия PDH]], [[Мультиплексирование]] (TDM собирает много PCM-каналов).
- **Соседи по уровню:** современные кодеки голоса (G.729, Opus, AMR) — пожимают PCM до 8–32 кбит/с.
- **Противопоставляется:** аналоговая передача — шум накапливается; PCM — нет.

## Подводные камни
- 64 кбит/с — это **только голос**. Музыка (CD, 44.1 кГц / 16 бит / 2 канала) = 1411 кбит/с в PCM.
- **Дискретизация ≠ Найквист-теорема о выборках:** часто путают. Найквист в §2.4.2 даёт максимальную скорость передачи данных через канал. Теорема выборок (Найквист-Шеннон) — **другая** теорема об условии восстановления.
- В современном VoIP (Skype, WhatsApp) PCM почти не используется — кодеки сжимают сильнее. Но 64 кбит/с — стандартный «эталон качества голоса».
