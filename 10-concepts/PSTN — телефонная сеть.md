# PSTN — телефонная сеть (Public Switched Telephone Network)

## TL;DR
Глобальная телефонная сеть с **коммутацией каналов**: при звонке от A к B заранее резервируется путь, и весь разговор идёт по нему. Иерархия коммутаторов: end office (АТС) → toll/tandem office → magistrals. На местной петле сигнал аналоговый; в магистрали — цифровой (PCM 64 кбит/с на голос). Историческая основа интернет-доступа (dial-up, ADSL); сегодня вытесняется VoIP.

## Какую проблему решает
До 1990-х все коммерческие сети связи (включая интернет-доступ для домохозяйств) опирались на телефонную инфраструктуру, потому что она у всех уже была. Понимание PSTN объясняет, **почему** многие принципы сетей сложились именно так: коммутация каналов и коммутация пакетов противопоставляются именно через PSTN vs интернет.

## Как работает

**Иерархия (упрощённая):**
1. **Абонент** — телефон в доме.
2. **Местная петля** (см. [[Местная петля]]) — медная пара до ближайшей АТС, обычно 1–10 км.
3. **End office** (центральная АТС, central office) — обслуживает тысячи абонентов в районе.
4. **Toll office** — между end office'ами в городе.
5. **Tandem / regional / international** — магистраль.

```mermaid
flowchart LR
  Phone --медная пара--> EO[End Office]
  EO --оптика/ИКМ--> TO[Toll Office]
  TO --магистраль--> TO2[Toll Office]
  TO2 --> EO2[End Office] --> Phone2
```

**На местной петле** сигнал **аналоговый** (3.1 кГц голосовая полоса). На end office стоит **кодек** (codec): преобразует аналог в **PCM** (Pulse Code Modulation) — 8000 выборок/с по 8 бит → 64 кбит/с на голосовой канал. Дальше всё цифровое и мультиплексируемое.

**Мультиплексирование:** T1 = 24 каналов × 64 кбит/с = 1.544 Мбит/с (США); E1 = 30 × 64 = 2.048 Мбит/с (Европа). Иерархия дальше: T3, E3 → SONET/SDH.

**Сигнализация:** установление соединения, набор номера, биллинг — отдельная сеть (SS7), не идёт по голосовому каналу.

## Пример
- **Звонок Москва → Нью-Йорк (1990):**
  1. Поднимаешь трубку → местная АТС открывает канал.
  2. Набираешь +1-212-... → SS7 несёт сигнализацию по другой сети.
  3. Резервируется путь: Москва end office → toll → международный шлюз → подводный кабель → США → New York.
  4. Голос: твой аналог → PCM 64 кбит/с → магистраль → обратно в аналог в Нью-Йорке.
  5. Положил трубку → канал освободился.

- **Dial-up интернет:** модем эмулирует голос, шумит в линию — провайдер с обратной стороны декодирует. Канал PSTN не знает, что это данные.

## Связи
- **Базируется на:** [[Коммутация пакетов vs коммутация каналов]] — PSTN — каноничный пример каналов.
- **Используется в:** [[Местная петля]] (физика последней мили), [[ADSL]] (поверх местной петли), [[Мультиплексирование]] (T1/E1 в магистрали).
- **Соседи по уровню:** интернет — параллельная сеть с коммутацией пакетов; VoLTE/VoIP — сегодняшняя телефония поверх IP.
- **Противопоставляется:** интернет — гибкая, без резервирования, но без гарантий QoS «из коробки».

## Подводные камни
- PSTN не «исчезла» в один момент — местные петли до сих пор используются для DSL, ADSL и факса. В развитых странах операторы планомерно переводят всё на IP-инфраструктуру.
- «Коммутация каналов» — логическая. Внутри современный PSTN packet-switched, но абоненту видно, что соединение «занято» от gehoben до повешено.
- В России многие провайдеры всё ещё используют местную петлю для ADSL — поэтому архитектура PSTN критична для понимания, почему в одних домах есть быстрый интернет, в других нет.
