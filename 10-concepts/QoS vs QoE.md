# QoS vs QoE

## TL;DR
**QoS** (Quality of Service) — измеримые **технические** параметры сети: задержка, jitter, потери, пропускная способность. **QoE** (Quality of Experience) — **воспринимаемое** качество для пользователя: «звонок звучал хорошо», «видео не лагает», «страница открылась быстро». QoS даёт инструменты, QoE — цель. Отличный QoS не гарантирует отличный QoE и наоборот.

## Какую проблему решает
Сетевые инженеры мерят то, что **легко** мерить (мс, %); пользователь оценивает то, что **чувствует**. Разделение терминов помогает не путать «сеть формально хорошая» и «пользователь доволен».

## Как работает

**Параметры QoS (измеряемые):**
- **Throughput / bandwidth** — мегабит в секунду.
- **Latency / RTT** — миллисекунды.
- **Jitter** — вариация задержки.
- **Packet loss** — % потерь.
- **Availability** — uptime.

**Метрики QoE (субъективные/композитные):**
- **MOS** (Mean Opinion Score) — оценка 1–5 для аудио (слушатели оценивают).
- **Page load time / Time to interactive** — для веба.
- **Rebuffering ratio** — для видео-стриминга.
- **Видеокачество** — VMAF, SSIM, PSNR (инструменты пытаются предсказать MOS).

**Чувствительность приложений:**
| Приложение | Чувствительно к |
|---|---|
| Голосовой звонок | latency (>150 мс плохо), jitter, loss |
| Видеоконференция | latency, jitter |
| Видеостриминг (YouTube) | bandwidth, rebuffering |
| Веб-сёрфинг | latency (DNS+RTT), bandwidth |
| Скачивание файла | bandwidth, loss |
| Игры | latency, jitter |

## Пример
**Wi-Fi-роутер показывает 100 Мбит/с (QoS отлично), но видеоконференция «прыгает».**
- 100 Мбит/с — пропускная способность downstream.
- Проблема может быть в **upstream** (для видео нужно ~5 Мбит up).
- Или в **jitter** (видео-кодек ломается при дрожании пакетов).
- **QoE = плохо**, хотя «скорость отличная».

**Обратно:** медленное соединение (QoS skromn) с правильным кодеком и буферизацией → пользователь не страдает (QoE OK).

## Подводные камни
- «100 Мбит/с — отлично!» — упрощение. Скорость без latency/jitter не предсказывает QoE для real-time.
- **Operator-измеряемый QoS** часто отличается от user-experienced QoE. Поэтому современные ISP мерят QoE через **probes** на конечных устройствах.
- Устранение QoS-проблем не всегда улучшает QoE: если узкое место в кодеке/CDN/устройстве, сеть здесь невинна.
