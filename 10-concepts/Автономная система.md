# Автономная система (AS)

## TL;DR
**AS** (Autonomous System) — единица организации интернета: сеть **под общим административным управлением** с **единой routing-policy**. Идентифицируется **ASN** (16- или 32-битным числом). Внутри AS — **IGP** (OSPF, IS-IS, RIP); между AS — **EGP** ([[BGP]]). ~75 000 AS в интернете на 2026 г. Главные единицы: **transit AS** (продают connectivity), **stub AS** (только потребляет), **multihomed** (несколько upstream'ов).

## Какую проблему решает
Интернет должен масштабироваться на миллионы маршрутизаторов. Один глобальный routing-protocol не работает. Решение: разделить интернет на **AS** — каждая управляется одним admin'ом, имеет свою policy. Внутри AS — детальный IGP. Между AS — BGP с агрегацией и policy. Это даёт O(log N) рост таблиц вместо O(N).

## Как работает

**ASN — идентификатор:**
- **16-битные** ASN (1-65535) — исторические, кончились 2010s.
- **32-битные** (RFC 6793) — расширенные.
- Выдаёт RIR (Regional Internet Registry): RIPE, ARIN, APNIC, LACNIC, AFRINIC.
- Private ASN range (64512-65534) — для внутреннего использования.

**Типы AS:**

| Тип | Описание | Пример |
|---|---|---|
| **Stub AS** | один upstream (single-homed); не транзит | малый ISP, корп. сеть |
| **Multihomed** | ≥ 2 upstream'а; не транзит | средний enterprise |
| **Transit AS** | продаёт transit другим AS | Tier-1, Tier-2 ISP |
| **Internet Exchange Point AS** | route-server (админ — IXP) | DE-CIX, MSK-IX |

**Иерархия маршрутизации в AS:**
- **IGP** (Interior Gateway Protocol): OSPF, IS-IS, EIGRP, RIP — **внутри** AS.
- **EGP** (Exterior Gateway Protocol): только **BGP-4** на практике — **между** AS.

**Что AS определяет:**
- **Routing-policy:** какие prefix'ы анонсировать кому.
- **Адреса:** AS владеет одним или несколькими IP-prefix'ами (allocated by RIR).
- **Peering decisions:** с кем связываться.

```mermaid
flowchart TB
  AS_RIPE[RIPE NCC<br/>(Регистратор)] -- выдаёт ASN --> AS1[AS 65000]

```

## Пример
**Российский провайдер AS-12389 (Ростелеком):**
- ASN 12389 (16-bit).
- Множество IP-prefix'ов в его обладании.
- Internal: IS-IS как IGP в magistrale.
- External: BGP с MTS (AS 8359), Mail.ru (AS 47764), и многими peer'ами.
- Его routing-policy: что анонсировать, кому, по каким COMMUNITY-tag'ам.

**Multi-AS organizations:** крупные компании могут иметь несколько ASN для разных регионов или legal entities.

## Связи
- **Базируется на:** [[Иерархическая маршрутизация]] (концепт), regulatory framework (IANA → RIR → AS).
- **Используется в:** [[BGP]] (главный пользователь), [[Транзитная сеть]] (экономика).
- **Соседи по уровню:** **IXP**, **PoP** — физические точки связи AS.
- **Противопоставляется:** **flat routing** (один global protocol) — невозможно на масштабе интернета.

## Подводные камни
- **AS path в BGP** — последовательность AS-IDs, через которые анонс прошёл. Длина — proxy для «расстояния», но не для latency.
- **AS hijacking:** атакующий анонсирует чужой prefix → перехват трафика. Защита — **RPKI** (Resource Public Key Infrastructure).
- **Looking glass servers** в каждой крупной AS позволяют заглянуть в их BGP-table — для debugging.
- **One AS = one entity** — миф. Современные крупные операторы могут иметь сотни internal-роутеров за одним ASN; один ASN не значит «один датацентр».
