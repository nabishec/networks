# Канальный уровень (Data Link Layer, L2)

## TL;DR
Уровень между физическим и сетевым. Берёт «**сырые биты**» от L1 и превращает их в **фреймы** — очерченные кадры с обнаружением ошибок и (опционально) надёжностью. Видит только **прямого соседа** по каналу (next hop, следующий узел). На разделяемых средах разбит на подуровни: **LLC** (Logical Link Control — надёжность поверх одного канала: ACK, нумерация, повторы) и **MAC** (Media Access Control — арбитраж: «кто говорит сейчас» на общей среде). См. [[Подуровень MAC]].

## Какую проблему решает
Физический уровень даёт поток битов с ошибками: помеха исказила бит — вы об этом не знаете. Сетевой уровень хочет видеть **пакеты** с гарантиями. Канальный уровень — мост: режет поток на фреймы, ставит на каждый контрольную сумму, при необходимости пересылает потерянные. Делает «ненадёжный канал» похожим на «поток фреймов с известным концом и началом».

## Как работает

**Четыре главные задачи:**

1. **Формирование фреймов** ([[Формирование фреймов|framing]]) — где границы между кадрами в потоке битов?
2. **Обнаружение и коррекция ошибок** — добавление избыточности (см. [[CRC — циклический избыточный код]], [[Корректирующие коды — Хэмминг]]).
3. **Управление потоком** — медленный получатель не должен утонуть в потоке отправителя.
4. **Надёжность** (опционально) — повторная передача потерянных, через [[ARQ]] и [[Скользящее окно]].

**Услуги, предоставляемые сетевому уровню:**
- **Без подтверждения, без соединения** (Ethernet) — выбросил фрейм, дальше не наше дело.
- **С подтверждением, без соединения** (Wi-Fi) — каждый фрейм ACK'ится.
- **С подтверждением и соединением** (HDLC) — установление, нумерация, разрыв.

```mermaid
flowchart LR
  Net[Сетевой уровень] -- пакет --> LLC[LLC]
  LLC -- надёжность над каналом --> MAC
  MAC -- арбитраж среды --> Phy[Физический уровень]
  Phy <-. провод/эфир .-> Phy2[Физический]
```

## Пример
- **Ethernet** в LAN: канальный — Ethernet-фрейм с MAC-source/dest, EtherType, payload, CRC. Без ACK на L2; надёжность — забота TCP.
- **Wi-Fi** в эфире: канальный — 802.11-фрейм с явным ACK (нет CSMA/CD, есть CSMA/CA + ACK). Потеря пакета в эфире — обычное дело, поэтому L2-ACK.
- **PPP** на dial-up/SONET: HDLC-подобный фрейм, опциональная нумерация и LCP.

## Связи
- **Базируется на:** [[Уровневая архитектура]] (L2 в стеке), [[Среда передачи данных]] (физический канал).
- **Используется в:** [[Ethernet — IEEE 802.3]], [[802.11 — Wi-Fi архитектура]], [[PPP]], [[DOCSIS]] — конкретные L2-протоколы.
- **Соседи по уровню:** [[Подуровень MAC]] — нижняя часть L2 на разделяемых средах; [[Сетевой уровень]] — выше.
- **Противопоставляется:** [[Сетевой уровень]] — end-to-end через много хопов; L2 — только один хоп.

## Подводные камни
- На каждом маршрутизаторе **L2-заголовок снимается и собирается заново** под среду следующего хопа. Поэтому Ethernet-MAC меняется от хопа к хопу, а IP-адрес — нет.
- L2 «надёжный» (Wi-Fi ACK, HDLC numbers) — это надёжность **per-hop**. End-to-end надёжность всё равно делает TCP. Ср. end-to-end argument в [[Цели проектирования сетей]].
- Деление L2 на LLC и MAC — стандарт IEEE 802. В Tanenbaum'е MAC подробно вынесен в гл. 4.
