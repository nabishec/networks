# Маршрутизатор (router)

## TL;DR
Сетевой узел, который **пересылает** пакеты между разными сетями, выбирая интерфейс по адресу назначения. Работает на сетевом уровне ([[Уровневая архитектура|L3]]): читает только служебную часть пакета с адресом получателя (IP-заголовок), не вскрывая «полезную нагрузку» вроде TCP/UDP. Не источник пользовательских данных — транзитная станция.

**Аналогия:** маршрутизатор — как сортировщик на почте. Смотрит индекс на конверте и решает, в какой мешок (направление) положить письмо. Содержимое письма ему не интересно.

## Какую проблему решает
Если у тебя 3 машины в одном сегменте Ethernet, они общаются напрямую. Но как только сетей становится **две и больше** (например, домашняя LAN и публичный интернет), нужен узел, знающий: «пакет с адресом X отправлять через интерфейс А, пакет с адресом Y — через интерфейс Б». Это и есть маршрутизатор. Его таблица маршрутов — компактное знание о топологии сети.

## Как работает
1. Получает фрейм на одном из интерфейсов.
2. Снимает канальный заголовок, читает IP-заголовок.
3. По IP-адресу назначения ищет в **таблице маршрутизации** запись с **longest prefix match** — той, у которой IP-адрес совпал с наибольшим числом начальных бит (см. [[IP-адресация и CIDR]]).
4. Уменьшает TTL, пересчитывает контрольную сумму, упаковывает в новый фрейм нужного канального уровня и отправляет в выбранный интерфейс.

Таблица маршрутов заполняется вручную (статика) или через [[OSPF]], [[BGP]] и др. — это [[Сетевой уровень]] в работе.

```mermaid
flowchart LR
  In[Входящий фрейм] --> L2[Снять канальный заголовок]
  L2 --> Lookup[Lookup IP-таблицы по prefix]
  Lookup --> Decr[TTL−=1, recompute checksum]
  Decr --> Out[Упаковать в новый фрейм]
  Out --> Iface[Отправить в выбранный интерфейс]
```

## Пример
Домашний Wi-Fi-роутер: интерфейс LAN (192.168.1.0/24) и WAN (публичный IP от провайдера). Когда телефон шлёт пакет к 8.8.8.8, роутер видит — адрес не из 192.168.1.0/24, значит, по дефолтному маршруту → WAN-интерфейс → провайдер. Провайдерский маршрутизатор — следующий в цепочке.

## Связи
- **Базируется на:** [[Хост]] — другая роль узла; [[IP-адресация и CIDR]] — таблица маршрутов работает по префиксам.
- **Используется в:** [[Интернет — архитектура]] — маршрутизаторы и есть «сеть»; [[OSPF]], [[BGP]] — протоколы заполнения таблиц.
- **Соседи по уровню:** [[Hub vs Switch vs Router vs Gateway]] — разные устройства разных уровней.
- **Противопоставляется:** **switch** работает на L2 (MAC), не открывает IP; **hub** просто повторяет биты на все порты.

## Подводные камни
- Домашний «роутер» обычно совмещает **router + switch + Wi-Fi AP + NAT + DHCP**. Это коммерческая упаковка; функции внутри разные.
- Маршрутизатор **не** хранит состояние TCP-соединения (это работа транспортного уровня на хосте). NAT — исключение, NAT-блок хранит mapping и фактически выходит за чистую L3-роль.
