# Подуровень MAC (Media Access Control)

## TL;DR
Нижняя часть канального уровня для **разделяемых** сред: эфир Wi-Fi, классический Ethernet на одном кабеле, кабельная сеть DOCSIS. Решает один центральный вопрос: **«кто говорит сейчас»**, чтобы передачи разных узлов не накладывались (коллизия). Tanenbaum выделил MAC в отдельную главу 4, потому что распределение разделяемой среды — большая самостоятельная тема.

## Какую проблему решает
На двухточечном канале (provider ↔ home через прямой кабель) проблем нет — каждая сторона может говорить когда хочет. Но как только в одной среде **много узлов** (Wi-Fi-эфир, шинный Ethernet 1980-х, кабельная сеть), нужен **арбитраж**: иначе узлы будут перебивать друг друга, и связь станет невозможной.

MAC — это про правила «кто и когда» в общей среде. От этих правил зависят пропускная способность, задержка и справедливость.

## Как работает

**Два класса решений ([[Проблема распределения канала]]):**

1. **Статическое распределение** — каждому свой выделенный кусок (TDMA, FDMA, CDMA). Просто, предсказуемо, но при пиковой нагрузке неэффективно (если кто-то молчит, его слот не передать другим).

2. **Динамическое (на основе арбитража)**:
   - **Random access:** [[ALOHA]], [[CSMA]], [[CSMA/CD]], [[CSMA/CA]] — узлы пробуют, при коллизии — отступают и повторяют.
   - **Безконфликтные:** [[Bit-map и token-passing]] — упорядоченный доступ, без коллизий.
   - **С ограниченной конкуренцией:** гибриды.

**Стандарты IEEE 802 формально делят L2 на:**
- **LLC** (802.2) — общая надёжность над одним каналом.
- **MAC** — конкретный для среды (802.3 для Ethernet, 802.11 для Wi-Fi и т.д.).

## Пример
- **Wi-Fi:** все устройства комнаты делят радиоэфир. MAC = [[CSMA/CA]] + [[NAV — Network Allocation Vector|NAV]] + [[RTS/CTS]] для борьбы с hidden terminal.
- **Классический Ethernet (10BASE5/2/T до switched):** общий шинный кабель. MAC = [[CSMA/CD]] + binary exponential backoff.
- **Bluetooth piconet:** master управляет TDM-слотами, slave'ы говорят только когда им разрешено.
- **DOCSIS:** upstream coax общий для всех домов узла. MAC = TDM с минислотами + явные запросы у CMTS.

## Связи
- **Базируется на:** [[Канальный уровень]] (MAC — его часть на разделяемых средах).
- **Используется в:** [[Ethernet — IEEE 802.3]], [[802.11 — Wi-Fi архитектура]], [[Bluetooth]], [[DOCSIS]] — каждый со своим MAC.
- **Соседи по уровню:** **LLC** — верхняя часть L2, общая надёжность.
- **Противопоставляется:** двухточечный канал (PPP, SONET) — там MAC не нужен, среда не разделяется.

## Подводные камни
- На «коммутируемом» Ethernet (с switch) каждый порт — фактически двухточечный, и MAC-арбитраж неактуален. Но протокольная роль MAC-адреса остаётся.
- В радиосредах MAC сложнее, чем в проводных: нельзя слушать передачу одновременно с отправкой → нет CSMA/CD, появляется CSMA/CA.
- Иерархия LLC/MAC — стандарт **IEEE 802**. В TCP/IP-модели обычно объединяют их в один «Link layer».
