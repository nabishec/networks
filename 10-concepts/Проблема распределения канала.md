# Проблема распределения канала

## TL;DR
Как разделить **общий канал** между несколькими передатчиками? Два подхода: **статическое распределение** (каждому свой выделенный кусок: TDMA, FDMA) или **динамическое** (узлы конкурируют за общий канал по протоколу). Статика проста, но плохо масштабируется при «бурстовом» трафике. Динамика эффективнее использует канал, но требует более сложного MAC.

## Какую проблему решает
N узлов в одной среде — это либо предсказуемая (но неэффективная) поделёнка, либо хитрый арбитраж. Tanenbaum в §4.1 ставит этот вопрос как фундаментальный для всей главы и для понимания, **почему** появились ALOHA/CSMA/Ethernet/Wi-Fi и т.д.

## Как работает

**Допущения для динамики (по Tanenbaum'у §4.1.2):**
1. **Независимый трафик:** N независимых станций, каждая с пуассоновским потоком фреймов.
2. **Один канал:** всё передаётся по одной общей среде.
3. **Возможность коллизий:** если две станции говорят одновременно, оба фрейма теряются.
4. **Непрерывное или дискретное время:**
   - Continuous: передача может начаться когда угодно.
   - Slotted: время делится на равные слоты, передача только в начале слота.
5. **Carrier sense / no carrier sense:** станции либо могут «прослушивать» канал перед отправкой, либо нет.

**Статическое распределение:**
- **TDMA** — каждый получает свой временной слот.
- **FDMA** — каждый получает свою частоту.
- **CDMA** — каждый получает свой код.
- **Плюсы:** просто, предсказуемо, гарантированная задержка.
- **Минусы:** **N → ∞ ⇒ задержка растёт линейно**, и при «бурстовом» трафике (типичный data) канал недоиспользован.

**Динамическое распределение:**
- **Random access** ([[ALOHA]], [[CSMA]] и потомки): отправляй, при коллизии повторяй с задержкой.
- **Безконфликтные** ([[Bit-map и token-passing]]): право говорить передаётся явно, нет коллизий.
- **Гибриды**: при низкой нагрузке — random; при высокой — упорядоченный.

## Пример
- **Сотовая сеть 2G GSM:** статический TDM (8 слотов на 200 кГц) + FDM (~124 канала). Подходит для предсказуемого голосового трафика.
- **Ethernet (классический):** [[CSMA/CD]] — динамический random access. Подходит для бурстового data-трафика.
- **Wi-Fi:** [[CSMA/CA]] + [[NAV — Network Allocation Vector|NAV]] — гибрид: contention для data, координация через RTS/CTS для длинных передач.
- **Token Ring:** [[Bit-map и token-passing]] — детерминистическая задержка, но низкая утилизация при низкой нагрузке.

## Связи
- **Базируется на:** [[Подуровень MAC]] — это его центральный вопрос; [[Мультиплексирование]] (статика — частный случай).
- **Используется в:** [[ALOHA]], [[CSMA/CD]], [[CSMA/CA]], [[Bit-map и token-passing]] — конкретные ответы.
- **Соседи по уровню:** [[Перегрузка сети]] — родственная проблема в большом масштабе.
- **Противопоставляется:** двухточечный канал — нет проблемы распределения вообще.

## Подводные камни
- На реальном «бурстовом» трафике статический FDMA/TDMA в среднем дают **ниже** утилизацию, чем динамический CSMA. Поэтому интернет на ALOHA-потомках, не на TDMA.
- «Эффективнее» зависит от метрики: пропускная способность, задержка, fairness, jitter — у каждого протокола свой профиль.
- Радио и провод имеют разные ограничения: в радио нет CSMA/CD (нельзя слушать и говорить одновременно), отсюда CSMA/CA.
