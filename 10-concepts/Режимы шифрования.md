# Режимы шифрования (block cipher modes)

## TL;DR
Block cipher (AES) шифрует **один блок** (128 бит) → нужен режим, чтобы зашифровать произвольный payload. Главные: **ECB** (плохо, не использовать), **CBC** (chained, нуждается в IV), **CTR** (counter, удобен для parallel), **GCM** (CTR + auth — современный AEAD-стандарт). **AEAD** (Authenticated Encryption with Associated Data) совмещает шифрование и authentication — используй его.

## Какую проблему решает
Понимать различие:
- ECB на типичных данных (картинка, исполняемый файл) **показывает структуру** — не secure.
- CBC уязвим к padding-oracle, требует MAC отдельно.
- GCM — современный AEAD, шифрование+authenticity в одном.

Выбор режима критичнее, чем выбор алгоритма.

## Как работает

### ECB (Electronic CodeBook) — НЕ использовать!
- Каждый блок — независимо `C_i = E(K, P_i)`.
- **Одинаковые plaintext-блоки → одинаковые ciphertext-блоки** → утечка структуры.
- Известный пример: «зашифрованный пингвин Tux» — силуэт виден.
- Используется только для очень коротких секретов (<= block size).

### CBC (Cipher Block Chaining)
- `C_0 = IV`, `C_i = E(K, P_i ⊕ C_{i-1})`.
- IV (Initialization Vector) — random, передаётся вместе с ciphertext.
- **Хорошо**: одинаковые plaintext → разные ciphertext.
- **Плохо**: padding-oracle атаки (POODLE, BEAST), не parallelize encryption.
- Устарел в TLS 1.3.

### CTR (Counter)
- `C_i = P_i ⊕ E(K, nonce || counter_i)`.
- Block cipher используется как **stream cipher**.
- **Plus:** parallelizable, random access.
- **Минус:** **никогда не повторять (key, nonce)** — иначе keystream collision → плохо.

### GCM (Galois/Counter Mode) — рекомендуемый
- CTR + аутентификационный тег (через GHASH multiplication в GF(2¹²⁸)).
- **AEAD** — encryption + authentication в одной операции.
- В TLS 1.3 — main mode (AES-GCM, ChaCha20-Poly1305).
- **Никогда не повторяй nonce** иначе catastrophic.

### Сравнение

| Режим | Конф. | Аутент. | Parallel | IV/Nonce | Использовать сейчас |
|---|---|---|---|---|---|
| **ECB** | плохая | нет | да | нет | **никогда** |
| **CBC** | OK | нет (нужен MAC отдельно) | dec only | да (random) | legacy |
| **CTR** | OK | нет | да | да (unique nonce) | требует MAC |
| **GCM** | OK | **да** | да | да (unique nonce) | **по умолчанию** |
| **CCM** | OK | да | да | да | WPA2 (вместо GCM в radio) |

## Пример
- **TLS 1.3:** AES-256-GCM или ChaCha20-Poly1305.
- **WPA2 (CCMP):** AES-128-CCM.
- **IPsec ESP:** AES-GCM.
- **Disk encryption:** **XTS** (Xor-Encrypt-Xor with Tweakable cipher) — special mode для random access на дисках.

## Связи
- **Базируется на:** [[DES и AES]] (block cipher → mode → шифр). 
- **Используется в:** TLS, IPsec, WPA2/3, SSH.
- **Соседи по уровню:** [[HMAC]] — отдельный auth-механизм; AEAD совмещает.
- **Противопоставляется:** ECB — anti-pattern.

## Подводные камни
- **GCM nonce reuse = catastrophe** — keystream collision → можно восстановить plaintexts. Random 96-bit nonce имеет small but nonzero collision-probability.
- **CBC + не authentic** — POODLE/BEAST. AEAD безопаснее.
- **XTS** — для disk, не для streams.
- **Длинные nonce / counter wraparound** — следить, чтобы (key, nonce) не повторился.
