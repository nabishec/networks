# Сетевой уровень (Network Layer, L3)

## TL;DR
Уровень между канальным и транспортным. Передаёт **пакеты** через **много** хопов и **много разных** L2-сетей от исходного хоста до конечного. Главные задачи: **адресация** (как назвать любого хоста в мире), **маршрутизация** (как найти путь), **управление перегрузкой**. Доминирующий протокол — **IP**.

## Какую проблему решает
Канальный уровень доставляет фрейм только до **прямого соседа**. Чтобы пакет долетел через интернет, нужно много хопов — и на каждом канал может быть разным (Ethernet → Wi-Fi → SONET → Wi-Fi). Сетевой уровень даёт **единый адресный план** (IP) и **алгоритмы маршрутизации**, чтобы любые два хоста в мире могли общаться.

## Как работает
- Адресация: каждый хост имеет [[IP-адресация и CIDR|IP-адрес]].
- Маршрутизаторы (см. [[Маршрутизатор]]) хранят **таблицу маршрутов** — `prefix → next-hop interface`.
- При получении пакета: longest-prefix match → next hop → новый L2-фрейм → отправка.
- Таблицы заполняются: вручную (статика), [[OSPF]] (внутри AS), [[BGP]] (между AS).

**Две парадигмы (см. [[Datagram subnet vs virtual-circuit subnet]]):**
- **Datagram** — каждый пакет независим, маршрут выбирается per-packet. IP.
- **Virtual circuit** — путь резервируется при установлении, потом все пакеты идут по нему. ATM, MPLS-LSP.

## Пример
GET wikipedia.org → IP-пакет с DST=208.80.154.224 → твой роутер выбирает default → провайдер → магистраль → дата-центр Wikipedia → веб-сервер. Около 10–15 хопов; на каждом IP-заголовок не меняется (кроме TTL и checksum), L2-фрейм меняется каждый раз.

## Связи
- **Базируется на:** [[Канальный уровень]] (пакеты идут через L2-фреймы), [[Уровневая архитектура]].
- **Используется в:** [[IPv4]], [[IPv6]], [[Маршрутизатор]] — конкретные реализации.
- **Соседи по уровню:** [[Транспортный уровень]] — выше; [[Канальный уровень]] — ниже.
- **Противопоставляется:** L2 — один хоп; L3 — много хопов через разные сети.

## Подводные камни
- L3 — единственный уровень, **обязательно общий** в интернете. Без IP интернет не интернет.
- В современном дата-центре часто **L3-fabric** (BGP/OSPF в каждом ToR) вместо L2 STP — масштабнее.
