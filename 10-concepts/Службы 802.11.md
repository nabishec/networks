# Службы 802.11

## TL;DR
802.11 предоставляет 9+ служб (services), разделённых на **station services** (связаны с конкретной STA) и **distribution services** (связаны с DS-сетью между AP). Главные: **authentication** (WPA2/3), **association/reassociation/disassociation** (присоединение к BSS), **distribution** (forwarding между AP), **MAC service data unit** (delivery), **integration** (мост в proвод), **transmit power control**, **dynamic frequency selection**.

## Какую проблему решает
Чтобы клиент работал в Wi-Fi-сети, нужны не только data-фреймы, но и **management**: найти AP, аутентифицироваться, ассоциироваться, обработать роуминг между AP, согласовать ёмкость и мощность. 802.11 явно специфицирует эти services.

## Как работает

### Station Services (для STA)
1. **Authentication** — STA доказывает identity AP. WPA2-PSK, WPA3-SAE, 802.1X (EAP).
2. **Deauthentication** — explicit прерывание ассоциации (DoS-vector!).
3. **Privacy / encryption** — WEP (устар.), WPA2-CCMP, WPA3-GCMP.
4. **MSDU Delivery** — собственно delivery data-фреймов.
5. **TPC** (Transmit Power Control, 802.11h) — ограничение мощности (для совместимости с радарами в 5 ГГц).
6. **DFS** (Dynamic Frequency Selection) — обнаружение радаров, переход на другой канал.

### Distribution Services (DS-related)
7. **Association** — STA присоединяется к конкретной BSS (одной AP).
8. **Reassociation** — STA переходит на другую AP (роуминг внутри ESS).
9. **Disassociation** — explicit выход из BSS.
10. **Distribution** — пересылка фрейма от source-AP к dest-AP через DS (обычно проводной Ethernet).
11. **Integration** — мост между 802.11 и не-802.11 LAN (e.g., Ethernet).

### Жизненный цикл подключения

```mermaid
sequenceDiagram
  STA->>AP: Probe Request
  AP->>STA: Probe Response (capabilities)
  STA->>AP: Authentication Request
  AP->>STA: Authentication Response
  STA->>AP: Association Request
  AP->>STA: Association Response (granted, AID)
  Note over STA,AP: 4-way handshake (WPA2/3)
  STA<->>AP: Data
  STA->>AP: Disassociation
```

**MFP** (Management Frame Protection, 802.11w) — защита auth/disauth-фреймов от подделки. WPA3 делает обязательным.

### Роуминг (reassociation)
- STA движется → сигнал текущей AP слабеет, соседней — растёт.
- STA шлёт **Reassociation Request** новой AP.
- Новая AP проверяет authentication у старой через DS.
- Поток data плавно переключается.

**Fast roaming** (802.11r): кеширует context для ускорения handover (важно для VoIP).

## Пример
**Сотрудник входит в офис, ноут видит ESS «Corp»:**
1. **Probe**: scan каналов, видит beacon.
2. **Authentication**: 802.1X через RADIUS-сервер; cert от ноута + cert от RADIUS.
3. **Association**: AP на 1-м этаже принимает.
4. **4-way handshake**: derive PTK.
5. Работа.
6. **Reassociation**: переходит в кафе (другая AP, тот же ESS) — fast roaming, без full re-auth.

## Связи
- **Базируется на:** [[802.11 — Wi-Fi архитектура]] (контекст BSS/ESS).
- **Используется в:** все Wi-Fi-операции; [[WPA2 и WPA3]] (auth), [[802.11 MAC — DCF]] (для data delivery).
- **Соседи по уровню:** **802.1X / EAP** — Enterprise auth-фреймворк, поверх 802.11.
- **Противопоставляется:** **открытый Wi-Fi** — без auth, без encryption (legacy).

## Подводные камни
- **Deauth-атака:** management-фреймы исторически без MFP → атакующий шлёт deauth от имени AP → STA disconnects. WPA3 + MFP закрывают.
- **Probe-floods:** STA постоянно probe'и SSID even when not connected → privacy leak (видна история). Современные ОС используют random MAC + minimize probe.
- **Reassociation timing:** если медленный auth — обрыв ноутбука при роуминге слышен в звонке.
