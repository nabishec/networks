# Сокеты Беркли (Berkeley sockets)

## TL;DR
Стандартный POSIX-API для сетевых соединений, происходит из **BSD UNIX** (1983). Базовые операции: `socket()` создать, `bind()` привязать к адресу/порту, `listen()` ждать входящих (server), `accept()` принять (server), `connect()` подключиться (client), `send()`/`recv()`, `close()`. Везде: Linux, macOS, BSD, Windows (Winsock — почти то же), Java/Python/Go-стандартные библиотеки — обёртки.

## Какую проблему решает
Каждое сетевое приложение нуждается в одном и том же: «открыть соединение, послать байты, получить байты». Без стандартного API каждая ОС/библиотека изобретала бы своё → несовместимость. Сокеты Беркли стали де-факто стандартом, перенесены практически везде.

## Как работает

**Жизненный цикл TCP-сервера:**
```c
int s = socket(AF_INET, SOCK_STREAM, 0);     // 1. создать сокет
bind(s, &addr, sizeof addr);                  // 2. привязать к (IP, port)
listen(s, BACKLOG);                           // 3. готов слушать
while (1) {
    int conn = accept(s, NULL, NULL);         // 4. блокирует, ждёт клиента
    recv(conn, buf, sizeof buf, 0);           // 5. читать
    send(conn, "hello", 5, 0);                // 6. писать
    close(conn);                              // 7. закрыть
}
```

**TCP-клиент:**
```c
int s = socket(AF_INET, SOCK_STREAM, 0);
connect(s, &server_addr, sizeof server_addr); // 3-way handshake
send(s, "GET / HTTP/1.1\r\n\r\n", 18, 0);
recv(s, buf, sizeof buf, 0);
close(s);
```

**UDP — проще:** нет `listen/accept/connect`; сразу `sendto/recvfrom` с указанием адреса.

**Параметры:**
- `AF_INET` (IPv4), `AF_INET6`, `AF_UNIX` (локальный IPC).
- `SOCK_STREAM` (TCP), `SOCK_DGRAM` (UDP), `SOCK_RAW` (свой L4).

**Дополнения:**
- `select()`, `poll()`, `epoll()` (Linux), `kqueue` (BSD/macOS) — мультиплексирование событий.
- `setsockopt()` — TCP_NODELAY, SO_REUSEADDR, SO_KEEPALIVE, и т.д.

## Пример
**Простой echo-сервер на Python:**
```python
import socket
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.bind(('0.0.0.0', 12345))
s.listen(5)
while True:
    conn, addr = s.accept()
    data = conn.recv(1024)
    conn.send(data)
    conn.close()
```

Под капотом — те же C-сокеты Беркли. Любой язык с сетью — обёртка над ними.

## Связи
- **Базируется на:** [[Транспортный уровень]] (это его API), POSIX.
- **Используется в:** все сетевые приложения; [[TCP]], [[UDP]] — реализуются под этим API.
- **Соседи по уровню:** **higher-level libs** (libcurl, requests) — обёртки сверху; epoll/kqueue — нижний уровень multiplexing.
- **Противопоставляется:** raw сетевые API (DPDK, kernel bypass) — обходят сокеты для low-latency.

## Подводные камни
- **Blocking vs non-blocking:** `recv()` без `O_NONBLOCK` блокирует thread. Для высокой нагрузки нужны epoll/async.
- **TCP send() ≠ доставлено:** `send()` возвращает «сколько байт принято в kernel buffer», не «сколько получил peer». Доставку гарантирует только TCP, не сам вызов.
- **TIME_WAIT** на закрытом соединении блокирует порт ~60–120 с. SO_REUSEADDR помогает.
- **Buffer sizes** (SO_SNDBUF, SO_RCVBUF) — критичны на LFN (см. [[Bandwidth-Delay Product]]).
