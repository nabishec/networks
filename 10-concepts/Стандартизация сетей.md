# Стандартизация сетей

## TL;DR
Сеть работает только тогда, когда узлы разных производителей **согласованы** в форматах и протоколах. Стандартизацию ведут несколько организаций: **IEEE** (Ethernet, Wi-Fi), **ISO** (модели, форматы), **ITU** (телеком), **IETF** (интернет-протоколы через RFC), **W3C** (веб). У каждой — свой процесс и сфера.

## Какую проблему решает
Если каждый вендор изобретёт свой Ethernet, оборудование не будет совместимо, рынок раздробится, пользователь проиграет. Стандарт — это публично доступная спецификация, которой следуют **все**, чтобы детали разных производителей встали друг к другу.

## Как работает

| Организация | Сфера | Способ работы |
|---|---|---|
| **IEEE** (Institute of Electrical and Electronics Engineers) | Ethernet (802.3), Wi-Fi (802.11), Bluetooth (802.15), MAC | технические комитеты, члены — инженеры/компании |
| **ISO** (International Org. for Standardization) | модели (OSI), форматы | национальные представительства, медленный комитетский процесс |
| **ITU** (International Telecommunication Union) | телеком (ADSL/V-серия, оптика, спутники, распределение спектра) | агентство ООН, правительства и операторы |
| **CableLabs / SCTE** | кабельные сети (DOCSIS) | консорциум кабельных операторов |
| **IETF** (Internet Engineering Task Force) | интернет-протоколы (IP, TCP, HTTP, DNS) | открытые рабочие группы, **«rough consensus and running code»**, выпускает **RFC** |
| **IAB** (Internet Architecture Board) | надзор IETF | назначает руководство IETF |
| **W3C** (World Wide Web Consortium) | веб (HTML, CSS, DOM) | вендоры браузеров |
| **3GPP** | сотовая связь (3G/4G/5G) | партнёрство региональных стандарт-органов |

**RFC (Request For Comments):** документы IETF. Любой может предложить (Internet-Draft), после обсуждения и реализации — публикация. RFC бывают: Standards Track, Informational, Experimental, Historic. Главные: RFC 791 (IP), 793 (TCP), 7540/9114 (HTTP/2, /3).

## Пример
- **IEEE:** Ethernet-фрейм формализован в 802.3. Любой вендор делает NIC по этому стандарту — оно работает с любым другим.
- **IETF:** TCP-handshake описан в RFC 9293 (2022, обновление RFC 793). Все TCP-стеки следуют ему — Linux, Windows, BSD.
- **W3C:** HTML5 — стандарт W3C, по которому браузеры рендерят страницы.

## Связи
- **Базируется на:** [[Компьютерная сеть]] — архитектурная необходимость общего языка.
- **Используется в:** [[Эталонная модель OSI]] (ISO), [[Эталонная модель TCP/IP]] (IETF), все протокольные заметки ссылаются на RFC.
- **Соседи по уровню:** open source — параллельный механизм де-факто стандартизации (если все используют Linux-стек, его поведение становится «стандартом»).
- **Противопоставляется:** проприетарные решения — работают только в одной экосистеме.

## Подводные камни
- Стандарт ≠ внедрение. RFC может существовать десятилетиями, не получив массовой поддержки (IPv6 — 1998, всё ещё не везде).
- IETF любит «rough consensus» — компромиссы могут быть некрасивыми, но получают **код**. ISO любит «полный консенсус» — получает красивые документы и (в случае OSI) громоздкие, медленные первые реализации, опоздавшие на рынок.
- IEEE и IETF иногда дублируются (есть и 802.1X для аутентификации в LAN, и RFC для EAP).
