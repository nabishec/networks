# Утопия — протокол 1 (Tanenbaum'а)

## TL;DR
**Учебный baseline-протокол:** один симплексный канал, нет потерь, нет ошибок, бесконечно быстрый получатель. Отправитель шлёт сразу всё, не ждёт ничего. Не реалистичен — отправная точка для обсуждения, через какие проблемы добавляются sequence numbers, ACK'и, флоу-control. Tanenbaum в §3.3.3 даёт его в коде на C для иллюстрации.

## Какую проблему решает
Прежде чем разбирать сложные ARQ-протоколы, нужно явно проговорить **исходные допущения**: где их слабости, как реальность их нарушает. Утопия — minimal протокол, демонстрирующий «что было бы, если бы ничего не нужно было решать». От него мотивируется Stop-and-Wait, GBN, SR.

## Как работает

**Допущения Утопии:**
- Канал симплексный (отправитель → получатель только).
- **Без ошибок** на канале.
- **Без потерь.**
- Получатель **бесконечно быстрый** (никогда не утопится в потоке).
- Сетевой уровень всегда готов отдавать пакет.

**Псевдокод:**
```c
void sender1(void) {
  frame s;
  packet buffer;
  while (true) {
    from_network_layer(&buffer);  // получить пакет
    s.info = buffer;               // упаковать в фрейм
    to_physical_layer(&s);         // отправить
  }
}

void receiver1(void) {
  frame r;
  event_type event;
  while (true) {
    wait_for_event(&event);        // только frame_arrival
    from_physical_layer(&r);       // получить
    to_network_layer(&r.info);     // отдать наверх
  }
}
```

**Что реальность нарушает:**
- Канал шумит → нужен CRC + retransmit ([[ARQ]]).
- Получатель медленнее → нужен flow control ([[Скользящее окно]]).
- Ошибки бывают → нужен seq number и detection.
- Дуплекс → piggyback ACK.
- Каждое нарушение → новый протокол:
  - Утопия → Stop-and-Wait (потери).
  - S&W → Go-Back-N (BDP).
  - GBN → Selective Repeat (массовые потери).

## Пример
Полезен только в учебнике. Реальный «утопичный» канал — короткий direct-attached cable между двумя машинами в идеальных условиях. Но даже там shutdown / restart требует synchronization.

## Связи
- **Базируется на:** [[Канальный уровень]] (минимальный протокол в нём).
- **Используется в:** учебная мотивация для [[Stop-and-Wait]], [[ARQ]], [[Go-Back-N]], [[Selective Repeat]].
- **Соседи по уровню:** **protocol.h** в Tanenbaum — фреймворк для всех учебных протоколов 1-6.
- **Противопоставляется:** реальные протоколы — все добавляют что-то сверху.

## Подводные камни
- **Только учебная роль.** В книге его обсуждение нужно для понимания, **зачем** нужны элементы реальных протоколов. Никто его не реализует в production.
- Tanenbaum использует Утопию как **«нулевую гипотезу»**: «если бы канал был идеален, было бы достаточно вот этого. Реальность не идеальна — поэтому…».
