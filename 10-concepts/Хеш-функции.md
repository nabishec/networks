# Хеш-функции (cryptographic hash functions)

## TL;DR
Функция `H(M) → digest` фиксированной длины (256 / 512 бит). **Свойства:**
- **Pre-image resistance:** по `H(M)` нельзя восстановить `M`.
- **Second pre-image resistance:** по `M1` нельзя найти `M2 ≠ M1` с тем же `H`.
- **Collision resistance:** не найти любые `M1 ≠ M2` с тем же `H`.

Современные стандарты: **SHA-2** (SHA-256, SHA-512), **SHA-3** (Keccak). MD5 и SHA-1 **сломаны** для collision-resistance.

## Какую проблему решает
Хэш — «отпечаток» данных. Нужен для:
- **Проверки целостности** (download checksum).
- **Подписи** (подписываем hash, а не сам файл — RSA быстрее на 256 битах).
- **Password storage** (Argon2, bcrypt — **специальные** «slow» hashes).
- **HMAC** для message authentication.
- **Blockchain** (proof-of-work, Merkle trees).

## Как работает

**Структура хэша:**
- Input: произвольной длины.
- Output: фиксированной (например, 256 бит для SHA-256).
- Detеrministic: тот же input → тот же output.

**Главные алгоритмы:**

| Алгоритм | Output | Status |
|---|---|---|
| **MD5** | 128 бит | сломан, collisions trivial |
| **SHA-1** | 160 бит | сломан (2017, SHAttered) |
| **SHA-256** (SHA-2) | 256 бит | стандарт |
| **SHA-512** (SHA-2) | 512 бит | стандарт, быстрее на 64-бит CPU |
| **SHA-3** (Keccak, 2015) | переменно | альтернатива на случай SHA-2-проблем |
| **BLAKE3** (2020) | 256 бит | очень быстрый |

**Размер output ↔ collision security:**
- N-bit hash → collision attack за 2^(N/2) операций ([[Birthday attack]]).
- 128 бит → 2^64 collision (доступно злоумышленнику!).
- 256 бит → 2^128 — astronomically unfeasible.

## Пример
**Проверка скачанного Linux ISO:**
- Сайт публикует SHA-256 checksum: `e1d33...`
- Скачали ISO → `sha256sum image.iso` → `e1d33...`
- Совпало → файл не повреждён в пути.
- (Для security-purposes ещё подпись от distro key.)

**Password storage:**
- НЕ просто `H(password)` — too fast → brute force.
- **Argon2id** (modern), **bcrypt** (legacy) — slow + salted.
- Хранить `(salt, slow_hash(password, salt))`.

**Signature:**
- File 100 МБ → SHA-256 → 32 байта.
- Подписать 32 байта RSA — secunds.
- Подписывать 100 МБ напрямую — невозможно (RSA ограничено block size).

## Связи
- **Базируется на:** криптографические primitives.
- **Используется в:** [[HMAC]] (auth), [[Цифровая подпись]] (sign), TLS, blockchain, integrity checks.
- **Соседи по уровню:** **HMAC, KDF** (Key Derivation Function — HKDF, PBKDF2) — производные функции.
- **Противопоставляется:** non-cryptographic hashes (CRC, MurmurHash) — не collision-resistant; для security не подходят.

## Подводные камни
- **MD5 / SHA-1 для security — НЕ использовать.** Для checksums non-security — ОК.
- **Length extension attack** на SHA-2 (Merkle-Damgård construction) → надо использовать **HMAC**, не плоский `hash(secret || message)`.
- **Password hashing ≠ general hashing.** Argon2/bcrypt должны быть медленными; SHA-256 — быстрый, легко brute-force паролей.
- **Quantum-safe?** Grover speedup ↓ effective security в 2 раза → SHA-256 → 128-bit security in quantum world. SHA-512 безопасен.
