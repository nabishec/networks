# Ethernet — IEEE 802.3

## TL;DR
Семейство стандартов IEEE 802.3 — самая распространённая проводная LAN-технология. Появился в Xerox PARC (Bob Metcalfe, 1973), стандартизирован в 1983. Прошёл путь от 10 Мбит/с по толстому коаксиалу до **400 Гбит/с** в дата-центрах. Базовая единица — Ethernet-фрейм с полями адресации (MAC), типа протокола (EtherType) и CRC-32.

## Какую проблему решает
Нужен дешёвый, простой, расширяемый стандарт LAN. Token Ring был сложнее и дороже; Arcnet — нишевый. Ethernet выиграл за счёт **простоты и стоимости**: одинаковый физический адрес, без центрального арбитра, на любой среде (коаксиал, витая пара, оптика).

## Как работает

**Структура фрейма (DIX/Ethernet II):**

```
+----------+--------+--------+----------+----------+----------+--------+
| Preamble | DST MAC| SRC MAC| EtherType| Payload  |   Pad    | CRC-32 |
| 8 байт   | 6 байт | 6 байт | 2 байта  | 46-1500  | если <46 | 4 байта|
+----------+--------+--------+----------+----------+----------+--------+
```

**Поля:**
- **Preamble** — 7 байт `10101010` + 1 байт SFD `10101011` для синхронизации.
- **Destination/Source MAC** — 48-битные адреса (см. [[MAC-адрес]]).
- **EtherType** — что в payload: `0x0800` IPv4, `0x0806` ARP, `0x86DD` IPv6, `0x8100` VLAN-тег и т.д.
- **Payload** — 46-1500 байт. Если данные < 46 — padding (нужно для CSMA/CD slot time).
- **CRC** — IEEE 802.3 CRC-32 для детектирования ошибок.
- **Inter-frame gap** — 12 байт тишины между фреймами.

**Минимальный фрейм** 64 байта (DST + SRC + Type + 46 payload + CRC) — physical requirement of CSMA/CD, см. [[CSMA-CD|CSMA/CD]].

**802.3 vs DIX:** IEEE 802.3 переопределил поле «Type» как «Length» и добавил [[Подуровень MAC|LLC]]-заголовок. Сегодня практически вся индустрия использует **DIX-формат с EtherType** (значения ≥ `0x0600` = тип, не длина). Tanenbaum (стр. 340) описывает эту историю.

**MAC-протокол:**
- **Классический (10BASE-2/5/T-half)**: [[CSMA-CD|CSMA/CD]] на разделяемой среде.
- **Коммутируемый**: каждый порт — двухточечная связь со switch'ем; CSMA/CD не нужен; full-duplex.

## Производительность классического Ethernet

Tanenbaum (стр. PDF 341–343) даёт формулу эффективности при загрузке N узлов и средней длине фрейма F бит:

$$ \text{Эфф} = \frac{1}{1 + 2BLe/cF} $$

где B — bandwidth, L — длина кабеля, e ≈ 2.718, c — скорость распространения. Член `2BLe/c` — пропорционален slot time × число конкурентных слотов.

**Slot time расширения:** пока идёт contention, в среднем уходит ~e ≈ 2.718 слотов до успеха одного узла.

| Размер фрейма | Эфф. (10 Мбит/с, 2.5 км кабель) |
|---|---|
| 64 байта (минимум) | ~30% |
| 1024 байта | ~85% |
| 1500 байт (max) | ~88% |

Вывод: **классический CSMA/CD Ethernet эффективнее на длинных фреймах**. Поэтому в современном switched Ethernet, где CSMA/CD не работает, эта формула не актуальна — но исторически она объясняет, почему минимальные фреймы плохи.

## Пример
**Domestic Wi-Fi роутер с проводным ноутбуком:**
- Ноут шлёт `GET / HTTP/1.1` к серверу.
- TCP-сегмент → IP-пакет → **Ethernet-фрейм**:
  - DST = MAC роутера (узнан через [[ARP]]).
  - SRC = MAC ноута.
  - EtherType = `0x0800` (IPv4).
  - Payload = IP-пакет.
- Роутер получил, прочитал IP, заменил Ethernet-фрейм на новый Wi-Fi-фрейм.

## Подводные камни
- **Jumbo frames** (до 9000 байт) — нестандарт; используются в дата-центрах, где все участники их поддерживают.
- В EtherType `0x0001`–`0x05DC` — старая «длина» по 802.3; ≥ `0x0600` — тип. Распознают по числу: больше 1500 → тип.
- **Half vs full duplex:** на хабе — полудуплекс, CSMA/CD активен; на switch'е — полнодуплекс, нет коллизий.
- В мире WAN используется **Carrier Ethernet** (MEF) — расширения для metro/long-haul поверх обычного 802.3.
