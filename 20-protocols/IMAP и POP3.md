# IMAP и POP3

## TL;DR
Два протокола **получения почты**: **POP3** (Post Office Protocol v3, RFC 1939) — скачать всё локально и (опционально) удалить с сервера. **IMAP** (Internet Message Access Protocol, RFC 9051) — почта живёт на сервере, клиенты синхронизируются с папками, флагами, search. POP3 — проще, IMAP — для multi-device.

## Какую проблему решает
SMTP доставляет письма к серверу получателя. Клиент (MUA) должен их **забрать**. До 1990-х был один компьютер per пользователь → POP3 (скачал, удалил с сервера). С появлением многих устройств (телефон + ноут + web) — POP3 неудобен → IMAP.

## Как работает

### POP3 (порт 110, или 995 для TLS)
**Команды:**
- `USER`, `PASS` — login.
- `STAT` — сколько сообщений и байт.
- `LIST` — список.
- `RETR n` — получить сообщение n.
- `DELE n` — пометить на удаление.
- `QUIT` — закрыть, удалить помеченные.

**Особенности:**
- **Stateful, простой.**
- По умолчанию **скачать и удалить**. Опция «leave on server» есть.
- Нет sync flags, нет folders на сервере, нет search.

### IMAP (порт 143, или 993 для TLS)
**Команды (более сложно):**
- `LOGIN`, `AUTHENTICATE`.
- `LIST` папок.
- `SELECT INBOX` — открыть папку.
- `FETCH n FLAGS|BODY|...` — получить часть/всё сообщения.
- `STORE n +FLAGS \Seen` — пометить прочитанным.
- `SEARCH FROM "alice@..."` — серверный поиск.
- `MOVE`, `COPY`, `EXPUNGE`.
- `IDLE` — push-уведомления (RFC 2177).

**Особенности:**
- Письма **остаются на сервере**.
- Папки, флаги (Seen, Answered, Flagged), синхронизируются.
- Можно **частично скачать** (только заголовки, потом тело по запросу).
- Синхронизация между устройствами автоматическая.

## Пример
**Gmail на телефоне + ноуте:**
- IMAP: оба видят одни и те же письма, прочитанные в одном — отмечены прочитанными в другом.
- Если бы был POP3: каждый клиент скачал свою копию → состояния разошлись бы.

**POP3 still useful:** для архивации (downloader скачивает всё в локальный mbox), embedded-устройств с малой памятью.

## Связи
- **Базируется на:** [[Email — архитектура]], [[TCP]].
- **Используется в:** все email-клиенты; Gmail, Outlook, Thunderbird, Apple Mail.
- **Соседи по уровню:** [[SMTP]] — для отправки; **JMAP** (RFC 8620) — современный JSON-based преемник IMAP.
- **Противопоставляется:** проприетарные протоколы (Exchange ActiveSync для Outlook).

## Подводные камни
- **IMAP IDLE** требует open TCP-connection — на mobile это батарея. Push (через provider's notification service) обычно лучше.
- **Большие mailbox'ы по IMAP** — некоторые клиенты медленны при синхронизации десятков тысяч писем.
- **POP3 + многоустройство** — не получится синхронизация. Использовать IMAP.
