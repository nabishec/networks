# Binary countdown (BRAP)

## TL;DR
**Безконфликтный** MAC-протокол на основе **побитового арбитража**. Каждая станция имеет двоичный ID. Хотят передавать → шлют свой ID **бит за битом**, начиная со старшего. Если станция шлёт 0, а в канале — 1 (от другой станции), она **отступает**. Победитель — станция с **наибольшим ID среди желающих**. Эффективность d/(d + log₂N), где d — длина данных. Применяется в **CAN-bus** (промышленная связь, авто).

## Какую проблему решает
Bit-map протокол требует резервного слота на каждую станцию — overhead растёт линейно с N. Token-passing требует кольцевую структуру и token-recovery. Binary countdown решает arbitrage за **log₂(N)** бит — компактно.

## Как работает

**Алгоритм (упрощённо):**
- Каждая станция имеет уникальный **k-битный ID** (k = log₂N).
- Хотят передавать → одновременно шлют ID-бит за битом.
- Канал реализует **OR**: если хоть один шлёт 1 — все слышат 1.
- Если станция шлёт **0**, но канал показывает **1** → знает, что есть «больший» ID соперник → **отступает**.
- В конце k бит остаётся **одна** станция — с наибольшим ID среди желавших.

**Пример** (Tanenbaum, илл. 4.8):
- 4 станции с ID: 0010, 0100, 1001, 1010.
- Хотят передавать: все 4.

| Бит # | OR на канале | Кто отступает |
|---|---|---|
| 1 (старший) | 1 | 0010, 0100 (шлют 0, видят 1) |
| 2 | 0 | — |
| 3 | 1 | 1001 (шлёт 0, видит 1) |
| 4 | 0 | — |

Победитель: 1010. Передаёт data.

**Эффективность:**
- 1 цикл = log₂N бит арбитража + d бит данных.
- Utilization = d / (d + log₂N).
- При 1024 станциях = 10 бит арбитража. Если data = 1000 бит → 99% utilization.

**Преимущества:**
- Безконфликтный (после арбитража точно один).
- Дешёвая аппаратная реализация (сравнение бит за бит).
- Приоритет: высший ID = более частый speaker.

## Пример
**CAN-bus** (Controller Area Network) — основной протокол в автомобилях, промышленной автоматике. Использует binary countdown для arbitrage между ECU (Engine Control Unit, ABS, transmission, и т.д.).
- Idle: канал recessive (1).
- ECU начинает передачу — dominant (0) overrides.
- Bitwise arbitration на 11-битном (или 29) message ID.
- Higher-priority message (lower ID) — wins.
- Используется во всех современных авто, медицинском, индустриальном оборудовании.

## Связи
- **Базируется на:** [[Проблема распределения канала]] (collision-free решение).
- **Используется в:** **CAN-bus**, RS-485 в industrial automation, некоторые academic protocols.
- **Соседи по уровню:** [[Bit-map и token-passing]] (другой collision-free), [[Adaptive tree walk]] (limited contention).
- **Противопоставляется:** ALOHA / CSMA — могут иметь коллизии; binary countdown — гарантия одного.

## Подводные камни
- **Требует физический канал, реализующий OR** (wired-OR на open-collector bus). На radio не работает напрямую.
- **Длина k = log₂N** — для миллионов станций уже 20+ бит overhead (vs data в 100 бит — 20% header).
- **Priority inversion:** if always-busy high-ID station — low-ID не получают шанс. CAN-bus это решает через **rate limiting** на ECU.
- В сетях data-центров не используется — there's обычно switched Ethernet.
