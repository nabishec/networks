# Leaky bucket

## TL;DR
Алгоритм **сглаживания** трафика: пакеты «вливаются» в виртуальную корзину фиксированного размера, «вытекают» с **постоянной** скоростью **r**. Бурст на входе сглаживается; на выходе — равномерный поток. Если корзина переполнилась — пакеты дропаются. **Не разрешает** бурст на выходе (в отличие от [[Token bucket]]).

## Какую проблему решает
В сетях с жёсткими требованиями к равномерности (старый ATM, traffic conditioning перед буфером маршрутизатора с малой ёмкостью) бурстовый трафик опасен. Leaky bucket даёт **детерминированно ровный** выход.

## Как работает

**Параметры:**
- **r** — leak rate (пакетов или байт/с).
- **B** — bucket size (буфер).

**Алгоритм:**
1. Пакет приходит → если в bucket'е есть место, добавляем; иначе drop.
2. Каждые `1/r` секунд из bucket'а удаляется один «единица» (пакет или N байт).

```
вход: ▓▓▓▓░░░░▓▓▓▓░░░░ (бурстовый)
                ↓
            [bucket]
                ↓ r=const
выход: ▓░▓░▓░▓░▓░▓░▓░▓░ (ровный)
```

**Сравнение с Token bucket:**

| | Leaky | Token |
|---|---|---|
| Output | строго r | до r в среднем, бурсты до B |
| Burst on output | нет | да |
| Idle: накопление кредита | нет | да |
| Цель | сглаживание | ограничение средней + burst |

## Пример
**Старый ATM-CBR (Constant Bit Rate)** требовал leaky bucket для регулярной отправки ячеек 53 байта.

**Современное применение:**
- VoIP: чтобы пакеты выходили строго через 20 мс — leaky bucket на эмиттере.
- API rate limit с requirement «равномерное распределение» (хотя чаще используют token bucket для гибкости).

## Связи
- **Базируется на:** [[Перегрузка сети]] (инструмент борьбы).
- **Используется в:** ATM CBR, real-time медиа-стримы, некоторые API rate-limit реализации.
- **Соседи по уровню:** [[Token bucket]] — допускает burst.
- **Противопоставляется:** именно эта пара. Leaky = жёсткий, Token = мягкий.

## Подводные камни
- Бурстовый трафик с малым B → много drop'ов на входе.
- Не подходит, когда у клиента есть «кредит идле»: leaky bucket выходит ровно r всегда.
- Чаще используется как **формальная модель** (для математической обоснованности) и для отдельных RT-сценариев; в реальных сетях преобладает token bucket.
