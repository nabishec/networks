# Link State Routing

## TL;DR
Каждый маршрутизатор: (1) измеряет состояние своих **прямых линков** к соседям; (2) **flood'ит** информацию (LSA — Link State Advertisement) **всем** маршрутизаторам зоны; (3) собирает полную топологию из чужих LSA; (4) запускает локально **Дейкстру** для своего sink tree. Быстрая конвергенция, нет count-to-infinity. Основа [[OSPF]] и IS-IS.

## Какую проблему решает
[[Distance Vector Routing]] страдает от [[Count-to-infinity problem]] и медленной конвергенции. Link State снимает это, разделяя задачу:
- **Распространение** информации — flooding надёжен и быстр.
- **Вычисление** маршрутов — локально Дейкстра, всегда сходится за один проход.

## Как работает

**5 шагов алгоритма (по Tanenbaum'у §5.2.5):**

1. **Discover neighbors:** при включении маршрутизатор отсылает HELLO-фреймы и слышит ответы — узнаёт соседей.
2. **Set link cost:** измеряет вес каждого линка (пинг, явная конфигурация, скорость).
3. **Build LSP:** формирует Link State Packet с (id, seq, age, list of (neighbor, cost)).
4. **Distribute LSP:** flood'ит во все интерфейсы. Каждый получатель проверяет seq — если новый, копирует и пересылает дальше; если старый, выбрасывает.
5. **Compute shortest paths:** на полученной полной топологии запускает Дейкстру → sink tree от себя.

**Sequence numbers + age:** защищают от старых LSA после рестарта маршрутизатора. После рестарта seq может быть слишком маленьким — поэтому LSA имеют **age**, через час истекают и заменяются.

## Пример
**OSPF area:** 20 маршрутизаторов.
- Каждый отправляет LSA каждые 30 минут (или сразу при изменении топологии).
- Все за ~1 секунду имеют одинаковую полную топологию.
- Каждый запускает Дейкстру → next-hop ко всем destination.
- Изменение топологии: новый LSA flood, все пересчитают за 1–5 с.

Сравнение: RIP в той же сети — 1–15 минут convergence + count-to-infinity. OSPF — секунды без петель.

## Связи
- **Базируется на:** [[Алгоритм Дейкстры]] (вычисление путей), [[Лавинная маршрутизация]] (распространение LSA).
- **Используется в:** [[OSPF]] (IETF), IS-IS (ISO, в магистралях провайдеров), MPLS-TE.
- **Соседи по уровню:** [[Distance Vector Routing]] — альтернативная парадигма.
- **Противопоставляется:** DV — LS требует больше памяти на маршрутизаторе (полная топология), но быстрее и без count-to-infinity.

## Подводные камни
- На очень больших сетях полная топология слишком велика → **areas** в OSPF, **levels** в IS-IS.
- LSA-flooding требует **надёжной** доставки (sequence numbers + ACK), иначе возникают inconsistent topology у разных маршрутизаторов.
- При изменениях много LSA одновременно → CPU spike на пересчёт Дейкстры. SPF throttling.
