# RSA (Rivest-Shamir-Adleman, 1977)

## TL;DR
**Первый практичный** асимметричный алгоритм. Стойкость — на сложности **факторизации** больших чисел. Ключи: пара (e, n) public + (d, n) private, где n = p·q (произведение двух больших простых). **Шифрование:** C = M^e mod n. **Расшифровка:** M = C^d mod n. **Подпись:** S = M^d mod n. Размеры: 2048-бит сегодня, 3072+ для long-term. Уязвим к квантовым компьютерам (Shor's algorithm).

## Какую проблему решает
Diffie-Hellman (1976) показал, что асимметричная криптография возможна — но только для key exchange. RSA (1977) дал **полноценную асимметричную пару**: можно зашифровать, расшифровать, подписать, проверить подпись.

## Как работает

**Генерация ключей:**
1. Выбрать два больших случайных простых числа p, q.
2. n = p·q.
3. φ(n) = (p−1)(q−1) — Euler's totient.
4. Выбрать e — coprime с φ(n) (обычно 65537).
5. d = e⁻¹ mod φ(n) — modular inverse.
6. **Public key:** (n, e). **Private key:** (n, d) (или вместо d — p, q, чтобы быстрее).

**Шифрование:** C = M^e mod n.
**Расшифровка:** M = C^d mod n.
**Подпись:** S = H(M)^d mod n (hash + sign).
**Проверка:** S^e mod n =? H(M).

**Почему работает:**
- Из теоремы Эйлера: M^φ(n) ≡ 1 mod n.
- e·d ≡ 1 mod φ(n), значит M^(e·d) ≡ M mod n.

**Безопасность:**
- Если знаешь n, нужно факторизовать → найти p, q → φ(n) → d.
- При n = 2048 бит — практически невозможно классическими алгоритмами.

## Пример
**TLS 1.2 с RSA:**
- Сервер имеет RSA-сертификат с public (n, e).
- Клиент: pre-master secret = random 48 байт.
- Клиент: encrypt с (n, e) → шлёт серверу.
- Сервер: decrypt с d → достал pre-master.
- Дальше symmetric AES.

В TLS 1.3 RSA-key-exchange **deprecated** в пользу ECDHE (perfect forward secrecy).

**Подпись:**
- Сервер подписывает hash сообщения своим d → клиент проверяет с e.

## Связи
- **Базируется на:** number theory (Euler, modular arithmetic), сложность факторизации.
- **Используется в:** SSL/TLS-сертификаты (исторически key exchange, сейчас в основном signatures), SSH-ключи, PGP, JWT-signing.
- **Соседи по уровню:** **ECDSA** (RFC 6979) — elliptic curve signatures, меньшие ключи; **EdDSA** (Ed25519) — современный fast-sign.
- **Противопоставляется:** [[Диффи-Хеллман]] — только key exchange, не подписи; ECC — меньшие ключи при той же security.

## Подводные камни
- **RSA-2048** ≈ 112 бит security; **RSA-3072** ≈ 128 бит. ECC-256 даёт ~128 бит — более компактно.
- **Padding критичен:** raw RSA без padding ломается. **OAEP** для encryption, **PSS** для signatures.
- **Quantum threat:** Shor's algorithm факторизует за полиномиальное время. Когда появятся достаточно мощные quantum-компьютеры (10+ лет?) — RSA умирает. **Post-quantum** алгоритмы (ML-KEM, ML-DSA) уже стандартизованы NIST.
- **RSA-1024 устарел** — не использовать.
