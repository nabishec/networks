# Token bucket

## TL;DR
Алгоритм **traffic shaping/policing**, ограничивающий **среднюю** скорость с разрешённым **бурстом**. Виртуальная «корзина» токенов наполняется со скоростью **r** токенов/с, имеет ёмкость **B**. Передача пакета размером N байт требует N токенов. Если есть — берём, передаём; нет — ждём (shaper) или drop (policer). Позволяет **накопить кредит** в тихие минуты и потратить его в пике.

## Какую проблему решает
Регулирование исходящей скорости пользователя/потока. **Простой ограничитель** (передавать ровно r байт/с) убивает любой бурст: пользователь ждёт даже если канал свободен. Token bucket даёт **гибкость**: накопленные токены = разрешение на короткие всплески, при сохранении средней r долгосрочно.

## Как работает

**Параметры:**
- **r** — token fill rate (токенов/с).
- **B** — bucket size (max накопленных токенов = max бурст).

**Алгоритм при отправке пакета size = N:**
1. Если tokens ≥ N: tokens -= N, передать.
2. Иначе: подождать пополнения (shaping) или дропнуть (policing).

**Динамика:**
- В состоянии покоя tokens растёт до B и держится.
- При активной передаче tokens падает; если падает до 0 — скорость = r.

```
tokens
  B ─┐
    │  ╱╲    ╱─ B (cap)
    │ ╱  ╲  ╱
    │╱    ╲╱  ← пользователь использует burst
  0 ─────────── time
```

**Token bucket vs Leaky bucket:**
| | Token bucket | Leaky bucket |
|---|---|---|
| Burst | разрешён до B | сглаживается, нет |
| Idle | накапливает кредит | нет |
| Output | бурстовый, но в среднем r | строго r |

## Пример
**Провайдер ограничивает абонента 100 Мбит/с со всплесками до 10 МБ:**
- r = 100 Мбит/с = 12.5 МБ/с.
- B = 10 МБ.
- Абонент сидит 10 минут не активен — bucket полный (10 МБ).
- Скачивает большой файл: первые 10 МБ — мгновенно (бурст из bucket); потом — 100 Мбит/с steady.
- Полезно: видеоконференции часто бурстовые, среднее низкое.

**В Linux:** `tc qdisc add ... tbf rate 100mbit burst 10mb` — token bucket filter.

## Связи
- **Базируется на:** [[Перегрузка сети]] (один из инструментов борьбы).
- **Используется в:** [[DiffServ]] (на edge для policing классов), traffic shaping в провайдерских сетях, AWS/cloud rate-limit'ах.
- **Соседи по уровню:** [[Leaky bucket]] — родственный алгоритм с другими свойствами.
- **Противопоставляется:** простой rate limit без burst — менее дружественен к бурстовому трафику.

## Подводные камни
- B = 0 эквивалентно строгому ограничению r — нет burst.
- Очень большой B → можно «накопить» долгое idle и затопить сеть на короткое время.
- На policer'е (drop при отсутствии токенов) **TCP видит потери** → срабатывает congestion control — нагрузка сама снижается.
