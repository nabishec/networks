# Корректирующие коды — Хэмминг

## TL;DR
**Расстояние Хэмминга** между двумя кодовыми словами = число позиций, в которых они отличаются. Чтобы **исправлять** до t ошибок, минимальное расстояние между допустимыми кодовыми словами должно быть ≥ **2t+1**. Чтобы только **детектировать** d ошибок — ≥ **d+1**. **Код Хэмминга (7,4)** — классический пример: 4 бита данных + 3 контрольных бит чётности → исправляет 1 ошибку.

## Какую проблему решает
Если послать данные «как есть», любая ошибка в канале их искажает. Можно **повторять** каждый бит 3 раза (тройное мажоритарное голосование) — но это 200% overhead. Хэмминг показал, что можно достичь исправления ошибок **гораздо эффективнее** через хитрое распределение проверочных бит. Это рождение теории кодирования (1950).

## Как работает

**Базовая идея:**
- Если два кодовых слова отличаются в **3 позициях** (расстояние 3), и канал внёс **1 ошибку** — изменённое слово ближе к исходному, чем к любому другому → можно исправить.
- При расстоянии 5 — исправим до 2 ошибок; и т.д.

**Формула:**
- Минимальное расстояние d_min для исправления **t** ошибок: **d_min ≥ 2t + 1**.
- Для детектирования **d** ошибок: **d_min ≥ d + 1**.

**Код Хэмминга (7, 4):**
- 4 бита информации (m = 4).
- 3 проверочных бита (r = 3) → расстояние 3 → исправление 1 ошибки.
- Кодовое слово 7 бит. Биты 1, 2, 4 (степени 2) — проверочные; остальные — данные.

> Tanenbaum в §3.2.1 (стр. PDF 255–256) приводит код Хэмминга **(11, 7)** как пример: 7 бит данных + 4 проверочных, контрольные позиции 1, 2, 4, 8. (7, 4) ниже — самый распространённый учебный вариант той же семьи; принципы те же.

**Расчёт проверочных бит** (позиции считаем от 1):
- p1 (позиция 1) проверяет позиции 1, 3, 5, 7 — те, у которых бит 0 индекса = 1.
- p2 (позиция 2) проверяет 2, 3, 6, 7 — бит 1 индекса = 1.
- p4 (позиция 4) проверяет 4, 5, 6, 7 — бит 2 индекса = 1.

**Расклад кодового слова (7,4) для данных `d1 d2 d3 d4 = 1011`:**

| Поз. | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|
| Содержит | p1 | p2 | d1 | p4 | d2 | d3 | d4 |
| Значение | ? | ? | 1 | ? | 0 | 1 | 1 |

- p1 = d1 ⊕ d2 ⊕ d4 = 1 ⊕ 0 ⊕ 1 = **0**
- p2 = d1 ⊕ d3 ⊕ d4 = 1 ⊕ 1 ⊕ 1 = **1**
- p4 = d2 ⊕ d3 ⊕ d4 = 0 ⊕ 1 ⊕ 1 = **0**

→ Кодовое слово: `0 1 1 0 0 1 1` = `0110011`.

**Декодирование:**
- На приёме считаем те же три XOR'а от полученных бит (включая проверочные).
- Получаем 3-битный «синдром» **s** = (s4, s2, s1). Если s = 0 — ошибок нет.
- Иначе **s в двоичном виде указывает позицию ошибки**. Это не магия: проверка pK сидит на позиции 2^K и охватывает позиции, у которых бит K индекса = 1, поэтому единственный «нарушенный» бит просвечивается в синдроме как двоичная запись своего индекса.

**Аналогия: «острова в океане».** Допустимые кодовые слова — острова на координатной сетке Хэмминга. Ошибка — шаг на 1 клетку. При расстоянии **3** между островами круги радиуса 1 вокруг них не пересекаются — приёмник всегда найдёт ближайший остров. При расстоянии 2 — круги касаются: одна ошибка детектируется (мы не на острове), но исправить нельзя.

**Пример исправления:** ошибка в 3-м бите → `0100011`. Считаем синдром: p1-набор = биты 1,3,5,7 = 0,0,0,1 → XOR = **1**; p2-набор = 1,0,1,1 → **1**; p4-набор = 0,0,1,1 → **0**. Синдром `011` (двоичное) = 3 → инвертируем 3-й бит → восстановлено `0110011`.

## Пример
- **Исторически:** Bell Labs, 1950. Хэмминг работал на компьютере с перфокартами, его раздражали ошибки на выходных, требующие ручной правки. Изобрёл код для автоматической коррекции.
- **Современно:** ECC-память сервера использует расширенные SEC-DED Hamming-подобные коды (Single Error Correction, Double Error Detection). Каждый 64-битный блок данных защищён 8 проверочными битами.
- **Низкоуровневые передачи:** Hamming(7,4) или (15,11) встроен в некоторые радиосвязи и telemetry-каналы.

## Связи
- **Базируется на:** теория линейных блочных кодов (см. n,m-параметризацию у Tanenbaum'а, стр. 254).
- **Используется в:** ECC RAM, DVD-кодирование, простые телеметрии. Базис для понимания более сложных кодов.
- **Соседи по уровню:** [[Код Рида-Соломона]] — для пакетных ошибок; [[LDPC и сверточные коды]] — современные мощные коды.
- **Противопоставляется:** [[CRC — циклический избыточный код]] — только **детектирует**, не исправляет.

## Подводные камни
- Хэмминг исправляет **независимые** одиночные ошибки. Пакетная ошибка (burst — несколько подряд) кладёт его на лопатки. Для этого — Reed-Solomon или интерливинг.
- На современном железе Хэмминг как корректирующий код почти не используется в каналах связи (заменён на LDPC, Polar). Но **в памяти** — стандарт.
- «Hamming weight» — вес кодового слова = число единиц в нём. Связано с расстоянием.
