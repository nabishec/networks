# CGNAT — Carrier-Grade NAT

## TL;DR
**Двухуровневый NAT на стороне оператора связи**: абоненты сидят в приватном `100.64.0.0/10` (RFC 6598), на пограничном маршрутизаторе оператора этот блок транслируется в небольшой пул реальных публичных IPv4. Результат: один публичный IP делят сотни абонентов, у конкретного пользователя нет «своего» внешнего адреса. Применяется массово после исчерпания IPv4 (2011 г.) — особенно в мобильных сетях. В РФ узел `198.18.x.x` (RFC 2544 для тестового трафика) на пути часто означает CGNAT-узел оператора и нередко совпадает с местоположением [[ТСПУ]].

## Какую проблему решает
Когда у оператора абонентов больше, чем доступных публичных IPv4 — нужно «уплотнить» использование адресов. Обычный [[NAT]] (NAT44) делает это в домашнем роутере: внутри LAN — `192.168.x.x`, снаружи — один публичный IP на семью. **CGNAT добавляет ещё один уровень**: абонентский роутер уже не получает публичный IP, а получает адрес из `100.64.0.0/10`. Дальше оператор делает второй NAT, выводя десятки–сотни абонентов в один внешний IP.

## Что подтверждает книга
- Tanenbaum: NAT появился из-за дефицита 32-битных адресов; provider может «выдавать» один и тот же IP по очереди (гл. 5, §5.7.2).
- Книга подробно разбирает классический NAT44 (CPE-уровень), но **отдельной главы про CGNAT нет** — это операторская практика, утвердившаяся уже после публикаций основных RFC.

## Подводные камни
- **Inbound-соединения невозможны:** нет «своего» публичного IP → port forwarding с домашнего роутера в интернет не работает; страдают P2P, gaming, видеозвонки без [[TURN-relay]].
- **IP shared с другими:** репутация (DDoS-чёрные списки, antifraud) одного абонента может рикошетить на сотни соседей.
- **Логирование** усложнено: для расследования нужны и timestamps, и порты — иначе нельзя сопоставить «кто из 500 абонентов сделал запрос».
- **РФ-маркер:** в трассировках мобильного интернета узлы `198.18.9.129`, `198.18.0.0/15` часто = CGNAT-шлюз оператора + точка вмешательства DPI/[[ТСПУ]] (см. src-05 в [[_sources]]).
- **IPv6** решает проблему радикально (адресов хватит всем) — но в России развёртывание медленное.

## Связи
- **Базируется на:** [[NAT]] — обычный NAT44 на CPE-роутерах.
- **Используется в:** мобильные операторы РФ (МТС/Билайн/Мегафон/Tele2), часть домашних провайдеров, особенно региональных.
- **Связано:** [[ТСПУ]] (часто на тех же узлах), [[Белые списки]] (CGNAT упрощает централизованную фильтрацию).
- **Противопоставляется:** [[IPv6]] — уход от NAT-этажерок; **публичный IPv4** (за доплату у провайдера) — нет CGNAT, есть inbound.
