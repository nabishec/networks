# DPI-фильтрация в РФ

## TL;DR
Сводная картинка всех DPI-методов, известных по 10 источникам (на 2026-05-02). DPI ([[Брандмауэр|deep packet inspection]]) в РФ применяется на **3 уровнях** трафика — DNS, IP, TLS. Методы: **SNI-фильтрация, IP/AS-blacklist, whitelist**, **active probing**, **session freezing**, **TLS-fingerprint analysis**, **DNS-spoofing**, **CGNAT-маршрутизация**. Реализуется через **ТСПУ** (РДП.РУ).

## Какую проблему решает (с точки зрения регулятора)
Дать государству **гранулярный** контроль над сетевым трафиком РФ без необходимости договариваться с провайдерами по каждому случаю.

## Как работает

### Уровни фильтрации

| Уровень | Метод | Заметка |
|---|---|---|
| L3 (IP) | IP-blacklist | прямой drop по dst |
| L3 (AS) | [[AS-level whitelist]] | блок целого AS (72 AS allowed, 391 пострадало) |
| L4 (TCP/UDP) | port filtering, [[Session freezing]] | замораживание после 16 KB |
| L7 (TLS) | [[SNI-фильтрация]] | чтение SNI в ClientHello |
| L7 (HTTP) | URL/header inspection | для unencrypted HTTP |
| L7 (DNS) | DNS-blacklist | dns-spoofing |

### Дополнительные методы

- **[[Active probing]]** — DPI сама подключается к suspicious IP проверять.
- **TLS fingerprint** (JA3/JA4) — классификация по handshake.
- **Whitelist DNS** (мобильные) — только Yandex 77.88.8.8 разрешён.
- **CGNAT-узел `198.18.x.x`** — маркер ТСПУ-карты.
- **TTL manipulation** — изменение TTL в обратном направлении (для оценки гетер).
- **ML-классификатор** (упоминается в src-10) — поведенческий анализ.

## Где ломается / почему может не работать (для регулятора)
- **Cloudflare/CDN** — блокировка целиком ломает легитимный бизнес.
- **WebRTC** — нельзя дропать без потери видеозвонков.
- **DNS** — нужен для всего.
- **Yandex/VK** — РФ-облака могут хостить обходные сервисы (см. [[Yandex API Gateway фронтинг]]).
- **Reality + Self-Steal** — внешне неотличимы от обычного HTTPS.
