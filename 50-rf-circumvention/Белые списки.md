# Белые списки (whitelist / default-deny)

## TL;DR
Архитектурный сдвиг РФ-фильтрации: от **blacklist** («блокируй конкретные сайты») к **whitelist** («пропускай только разрешённые»). По умолчанию — **запрещено всё**. На мобильных операторах разрешено ~0.14% IP-адресов (src-01); на проводных — экспансия идёт с конца 2025. Срывается в whitelist-модели всё, что не Yandex/Mail.ru/Госуслуги/Сбербанк и т.п.

## Какую проблему решает (с точки зрения регулятора)
- Blacklist — постоянная гонка: новый сайт = новая блокировка.
- Whitelist — **обратное**: новый сайт = по умолчанию недоступен. **Гораздо** жёстче.
- Решение «гонки вооружений» через смещение default'а.

## Как работает

**Уровни whitelist'а:**
1. **DNS-whitelist** (src-05): только определённые DNS-resolvers (Yandex 77.88.8.8) разрешают резолв; Cloudflare/Google могут не отвечать или давать NXDOMAIN.
2. **IP-whitelist** (L3): пакет с dst=non-whitelist-IP — drop в transit.
3. **AS-whitelist** (L3+): пакет с dst=AS-not-in-list — drop. Покрывает миллионы IP одним правилом. См. [[AS-level whitelist]].
4. **SNI-whitelist** (L7): TLS-handshake с SNI not-in-list — drop / freeze. См. [[SNI-фильтрация]].
5. **Comb. (DNS+IP/Transport)**: src-05 — двухуровневая на мобильных.

**Эволюция (по статьям):**
- 2024 (src-08): blacklist + точечные блокировки.
- 2025 февраль (src-09): whitelist для 72 AS, коллатеральный ущерб 391 AS.
- 2025 декабрь (src-06): «эпоха белых списков», конфиги массово ломаются.
- 2026 апрель (src-01, src-03): мобильные операторы full-whitelist, разрешено ~0.14%.

**Откуда whitelist:**
- Список доменов и AS — публичные ресурсы (cheburcheck.ru):  Yandex, VK, Mail.ru, Sberbank, Госуслуги, и т.д.
- Иногда — индивидуально по сегменту (только нужные для оператора-партнёра).

## Где ломается / почему может не работать (с точки зрения обходчика)
Стратегии, обходящие whitelist:
1. **Использовать whitelist-IP как первый hop** — Yandex Cloud (см. [[Yandex API Gateway фронтинг]]), VK Cloud, Yandex CDN.
2. **CDN-фронтинг** — Cloudflare ходит по IP, который **может** быть в whitelist (но src-09 показывает: AS 13335 Cloudflare попадает под удар).
3. **WebRTC** — Zoom/Discord обычно в whitelist.
4. **DNS-туннели** — DNS-resolution часто допускается.
